# 常见问题

> **安装和使用中的常见问题解答**

---

## 安装问题

### Q1: pip install 失败怎么办？

**A:** 尝试以下解决方案：

```bash
# 方案1: 使用国内镜像
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple forgeai

# 方案2: 升级pip
pip install --upgrade pip
pip install forgeai

# 方案3: 使用conda
conda install -c conda-forge forgeai
```

### Q2: Python版本不兼容怎么办？

**A:** ForgeAI 需要 Python 3.8+，检查版本：

```bash
python --version
# 需要 3.8 或更高

# 如果版本过低，安装新版本Python
# Windows: 从 python.org 下载安装
# macOS: brew install python@3.9
# Linux: sudo apt-get install python3.9
```

### Q3: 依赖安装失败怎么办？

**A:** 分步安装依赖：

```bash
# 安装核心依赖
pip install chromadb
pip install openai
pip install jieba

# 安装其他依赖
pip install -r requirements.txt
```

## 配置问题

### Q4: API Key 如何获取？

**A:** 根据LLM提供商获取：

**OpenAI:**
1. 访问 https://platform.openai.com/api-keys
2. 创建新的API Key
3. 复制到 `config.json` 的 `llm.api_key`

**Kimi:**
1. 访问 https://platform.moonshot.cn/
2. 注册并获取API Key
3. 设置 `llm.provider: "kimi"`

**DeepSeek:**
1. 访问 https://platform.deepseek.com/
2. 注册并获取API Key
3. 设置 `llm.provider: "deepseek"`

### Q5: 如何配置多个LLM？

**A:** 在 `config.json` 中配置：

```json
{
  "llm": {
    "providers": {
      "openai": {
        "api_key": "your-openai-key",
        "base_url": "https://api.openai.com/v1",
        "model": "gpt-4o-mini"
      },
      "kimi": {
        "api_key": "your-kimi-key",
        "base_url": "https://api.moonshot.cn/v1",
        "model": "moonshot-v1-8k"
      }
    },
    "default": "openai"
  }
}
```

### Q6: 配置文件在哪里？

**A:** 配置文件位置：

- **项目配置**: `项目目录/config.json`
- **全局配置**: `~/.forgeai/config.json`
- **环境变量**: `FORGEAI_API_KEY`

优先级: 环境变量 > 项目配置 > 全局配置

## 使用问题

### Q7: 命令找不到怎么办？

**A:** 检查安装和PATH：

```bash
# 检查是否安装
pip show forgeai

# 检查命令位置
which forgeai  # Linux/macOS
where forgeai  # Windows

# 如果找不到，重新安装
pip install -e .
```

### Q8: 项目初始化失败怎么办？

**A:** 检查目录权限和名称：

```bash
# 检查目录权限
ls -la

# 使用英文项目名
forgeai init --name "my-novel" --genre "fantasy"

# 检查磁盘空间
df -h
```

### Q9: 写作命令卡住怎么办？

**A:** 可能的原因和解决方案：

**原因1: API调用超时**
```bash
# 增加超时时间（在config.json中）
{
  "llm": {
    "timeout": 60
  }
}
```

**原因2: 网络问题**
```bash
# 检查网络连接
ping api.openai.com

# 使用代理（如果需要）
export HTTP_PROXY=http://your-proxy:port
export HTTPS_PROXY=http://your-proxy:port
```

**原因3: API Key无效**
```bash
# 验证API Key
curl https://api.openai.com/v1/models \
  -H "Authorization: Bearer your-api-key"
```

### Q10: RAG检索没有结果怎么办？

**A:** 检查索引状态：

```bash
# 检查索引文件
ls -la .forgeai/index.db
ls -la .forgeai/vectors/

# 重建索引
forgeai rebuild-index

# 检查索引内容
forgeai search "测试" --verbose
```

## 审查问题

### Q11: Agent审查全部失败怎么办？

**A:** 检查章节内容和格式：

```bash
# 检查章节文件
cat 4-正文/第1章.md

# 确保格式正确
# 第1章.md 应包含章节标题和正文
# 例如：
# # 第1章 觉醒
# 
# 正文内容...
```

### Q12: 如何跳过某些Agent？

**A:** 在命令中指定：

```bash
# 只运行特定Agent
forgeai check review 1 --agents consistency,continuity

# 跳过某些Agent
forgeai check review 1 --skip-agents ooc,pacing
```

### Q13: 审查报告在哪里？

**A:** 审查报告位置：

```
5-审查/
├── 第1章-审查报告.md      # 综合审查报告
├── 第1章-一致性检查.md    # 单个Agent报告
├── 第1章-OOC检测.md
└── ...
```

## 性能问题

### Q14: 实体提取太慢怎么办？

**A:** 优化实体提取：

```json
// 在 config.json 中配置
{
  "entity_extraction": {
    "enabled": true,
    "cache": true,
    "batch_size": 100,
    "use_ner": false  // 关闭NER，使用词典匹配
  }
}
```

### Q15: RAG检索太慢怎么办？

**A:** 优化RAG配置：

```json
{
  "rag": {
    "chunk_size": 500,      // 增大分块大小
    "chunk_overlap": 50,    // 减小重叠
    "top_k": 5,             // 减少返回数量
    "use_reranker": false   // 关闭Reranker
  }
}
```

### Q16: 内存占用过高怎么办？

**A:** 优化内存使用：

```json
{
  "performance": {
    "max_memory": "2GB",
    "clear_cache_interval": 100,
    "use_streaming": true
  }
}
```

## 数据问题

### Q17: 如何备份项目数据？

**A:** 备份方法：

```bash
# 方法1: Git备份
git add .
git commit -m "backup"
git push

# 方法2: 手动备份
cp -r 项目目录 项目目录_backup_$(date +%Y%m%d)

# 方法3: 导出数据
forgeai export --output backup.json
```

### Q18: 如何恢复数据？

**A:** 恢复方法：

```bash
# 方法1: Git恢复
git checkout HEAD~1

# 方法2: 从备份恢复
cp -r 项目目录_backup_20260420 项目目录

# 方法3: 从导出恢复
forgeai import --input backup.json
```

### Q19: 如何迁移项目？

**A:** 迁移步骤：

```bash
# 1. 导出项目
forgeai export --output project.json

# 2. 复制到新位置
cp project.json /new/location/

# 3. 在新位置导入
cd /new/location/
forgeai import --input project.json
```

## 高级问题

### Q20: 如何自定义Agent？

**A:** 创建自定义Agent：

```python
# 在 system/agents/ 创建 my_agent.py
from forgeai.agents import BaseAgent

class MyAgent(BaseAgent):
    name = "my-agent"
    
    def review(self, chapter, text):
        # 自定义审查逻辑
        issues = []
        # ...
        return issues
```

然后在 `config.json` 中注册：

```json
{
  "agents": {
    "custom": ["my_agent.MyAgent"]
  }
}
```

### Q21: 如何使用本地模型？

**A:** 配置本地模型：

```json
{
  "llm": {
    "provider": "ollama",
    "base_url": "http://localhost:11434/v1",
    "model": "llama3"
  }
}
```

确保Ollama已安装并运行：

```bash
# 安装Ollama
curl -fsSL https://ollama.com/install.sh | sh

# 运行模型
ollama run llama3
```

### Q22: 如何扩展功能？

**A:** 使用插件系统：

```python
# 创建插件
# system/plugins/my_plugin.py

from forgeai.plugins import Plugin

class MyPlugin(Plugin):
    name = "my-plugin"
    
    def on_chapter_written(self, chapter, text):
        # 章节写入后的处理
        pass
    
    def on_review_complete(self, chapter, result):
        # 审查完成后的处理
        pass
```
