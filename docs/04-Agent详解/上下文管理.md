# 上下文管理 Agent (context-agent)

## 概述

上下文管理Agent负责维护创作过程中的各种上下文状态，包括角色状态、情节状态、悬念状态、预期状态等，确保各Agent能够获取准确的上下文信息。

## 核心功能

### 1. 角色状态管理
追踪角色的当前状态和变化：

```
角色状态包含:
- 位置：角色当前所在地
- 情绪：角色的情绪状态
- 关系：与其他角色的关系
- 知识：角色已知的信息
- 目标：角色的当前目标
- 状态：角色的物理状态（健康、装备等）

状态更新:
每当角色发生重要变化，上下文管理Agent会更新状态
```

### 2. 情节状态管理
追踪情节的发展状态：

```
情节状态包含:
- 主线情节：主要冲突的进展
- 支线情节：次要情节的进展
- 伏笔状态：已埋下的伏笔
- 悬念状态：未解决的悬念
- 冲突状态：当前活跃的冲突

情节追踪:
- 已完成情节
- 进行中情节
- 待开启情节
```

### 3. 悬念状态管理
管理悬念的创建、推进和解决：

```
悬念生命周期:
1. 创建：悬念被引入
2. 推进：悬念获得新线索
3. 延迟：悬念暂时搁置
4. 解决：悬念真相大白
5. 反转：悬念结果出人意料

悬念状态:
{
  "id": "悬念001",
  "content": "神秘人的身份",
  "created_chapter": 5,
  "current_status": "推进中",
  "clues": ["线索1", "线索2"],
  "resolved_chapter": null
}
```

### 4. 预期状态管理
管理读者预期的建立和满足：

```
预期类型:
- 情节预期：读者期待某事件发生
- 情感预期：读者期待某种情感体验
- 关系预期：读者期待角色关系发展
- 揭秘预期：读者期待真相揭露

预期管理:
- 建立预期 → 延迟满足 → 满足/反转
- 预期强度：弱/中/强
- 预期状态：建立/延迟/满足/反转
```

### 5. 冲突状态管理
追踪冲突的发展和解决：

```
冲突状态:
- 类型：人际/内心/环境/命运
- 强度：1-10
- 阶段：建立/升级/高潮/解决
- 涉及角色：[角色列表]
- 开始章节：N
- 解决章节：M (null表示未解决)

冲突追踪:
- 活跃冲突：当前正在进行的冲突
- 已解决冲突：已经结束的冲突
- 潜在冲突：即将爆发的冲突
```

### 6. 章节上下文
维护每章的上下文信息：

```
章节上下文:
{
  "chapter": 5,
  "summary": "李明发现神秘信件",
  "characters": ["李明", "王芳"],
  "locations": ["北京", "公司"],
  "events": ["发现信件", "开始调查"],
  "suspenses": ["悬念001"],
  "conflicts": ["冲突002"],
  "hooks": ["章节结尾钩子"],
  "mood": "悬疑"
}
```

## 使用示例

### 基本使用
```bash
# 查看当前上下文
forgeai context view

# 查看特定角色的状态
forgeai context view --character "李明"

# 查看情节状态
forgeai context view --plot

# 查看悬念状态
forgeai context view --suspense
```

### 高级选项
```bash
# 更新角色状态
forgeai context update --character "李明" --location "上海"

# 添加悬念
forgeai context add-suspense --content "神秘人的身份"

# 解决悬念
forgeai context resolve-suspense --id "悬念001"

# 建立预期
forgeai context add-expectation --content "李明会揭开真相"

# 导出上下文
forgeai context export --format json

# 重置上下文
forgeai context reset
```

## 配置选项

```yaml
context_agent:
  # 上下文数据存储路径
  context_path: ".forgeai/context.json"
  
  # 是否自动更新上下文
  auto_update: true
  
  # 状态追踪粒度
  tracking_granularity: "high"
  
  # 是否追踪角色情绪变化
  track_mood: true
  
  # 是否追踪角色知识
  track_knowledge: true
  
  # 是否追踪悬念状态
  track_suspense: true
  
  # 是否追踪预期状态
  track_expectation: true
  
  # 上下文历史保留
  keep_history: true
  
  # 历史保留深度
  history_depth: 10
```

## 与其他Agent的协作

### 为其他Agent提供上下文

```
OOC检测:
→ 查询角色当前状态（位置、知识、能力）
→ 验证行为是否符合状态

连贯性检查:
→ 查询情节状态
→ 验证事件是否连贯

追读力检查:
→ 查询悬念状态
→ 评估悬念密度和推进

高潮点检查:
→ 查询冲突状态
→ 评估高潮铺垫和余波

节奏检查:
→ 查询章节上下文
→ 评估节奏变化
```

### 接收其他Agent的更新

```
创作Agent:
→ 新章节完成
→ 更新角色状态、情节状态、章节上下文

审查Agent:
→ 发现问题
→ 标记上下文异常

用户:
→ 手动修正
→ 更新上下文状态
```

## 常见问题

### Q: 上下文数据如何持久化？
A: 上下文数据存储在 `.forgeai/context.json` 文件中，每次更新都会自动保存。可以选择保留历史版本，便于回溯。

### Q: 上下文如何自动更新？
A: 当启用 `auto_update` 时，系统会分析新创作的内容，自动提取角色状态变化、情节推进、悬念创建等信息，更新上下文。

### Q: 如何处理上下文冲突？
A: 当检测到上下文冲突（如角色位置矛盾）时，系统会发出警告，并提供冲突详情。用户可以选择：
- 接受新状态（覆盖旧状态）
- 保持旧状态（忽略新状态）
- 手动合并

### Q: 多线叙事的上下文如何管理？
A: 系统会为每条故事线维护独立的上下文，并在线索交汇时合并上下文。可以选择查看单线上下文或全局上下文。

## 最佳实践

1. **定期检查上下文**：每完成几章，检查上下文状态是否准确
2. **及时更新状态**：重要事件发生后，及时更新上下文
3. **验证上下文一致性**：运行其他Agent前，确保上下文准确
4. **保留历史版本**：启用历史保留，便于回溯和调试
5. **导出上下文备份**：重要节点导出上下文，防止数据丢失
