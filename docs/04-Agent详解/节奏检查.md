# 节奏检查 Agent (pacing-checker)

## 概述

节奏检查Agent负责评估叙事节奏的合理性，检测过快、过慢、单调等问题，确保故事发展速度与内容匹配，维持最佳阅读体验。

## 审查维度

### 1. 叙事速度
检测情节推进的速度：

```
速度类型:
- 快节奏：冲突密集，情节跳跃，适合高潮、战斗
- 中节奏：平稳推进，适合日常、成长
- 慢节奏：细腻描写，适合情感、氛围

问题示例:
战斗场景使用慢节奏，大量心理描写
→ 建议：加快节奏，减少心理描写，增加动作描写
```

### 2. 信息密度
检测单位篇幅内的信息量：

```
信息密度计算:
信息量 = 新信息数量 / 篇幅

问题示例:
第5章：5000字，仅推进1个情节点 (密度过低)
第6章：2000字，推进5个情节点 (密度过高)

建议：保持均衡的信息密度，避免拖沓或匆忙
```

### 3. 场景长度
检测场景描写的长度是否恰当：

```
场景类型与建议长度:
- 战斗场景：1500-3000字
- 对话场景：1000-2000字
- 日常场景：800-1500字
- 情感场景：1200-2500字
- 过渡场景：300-800字

问题示例:
战斗场景写了8000字，读者疲劳
→ 建议：拆分为多个小战斗，穿插其他内容
```

### 4. 描写比例
检测不同类型描写的比例：

```
描写类型:
- 动作描写：推动情节
- 对话描写：展现性格、推进关系
- 心理描写：深化角色
- 环境描写：营造氛围
- 说明描写：提供背景

建议比例 (根据场景类型调整):
动作:对话:心理:环境:说明 = 3:3:2:1:1
```

### 5. 段落节奏
检测段落长度和结构：

```
段落节奏问题:
- 全是长段落：阅读疲劳
- 全是短段落：碎片化，缺乏深度
- 段落长度单调：缺乏节奏感

建议：长短段落交替，形成节奏感
```

### 6. 章节节奏
检测章节间的节奏变化：

```
章节节奏曲线:
章节:  1  2  3  4  5  6  7  8
节奏: 快 中 中 快 慢 中 快 快

问题识别:
- 节奏单调：连续多章节奏相同
- 节奏突变：相邻章节节奏差异过大
```

## 评估指标

### 节奏评分
```
评分维度:
- 速度适宜性 (0-10)
- 密度合理性 (0-10)
- 描写均衡性 (0-10)
- 段落节奏感 (0-10)
- 整体流畅度 (0-10)

综合评分 = 加权平均
```

### 节奏曲线
```
可视化展示:
章节节奏变化曲线
识别问题区域:
- 节奏过快区
- 节奏过慢区
- 节奏单调区
```

## 问题示例

### 示例1：节奏过慢
```
【问题】第5章
类型：叙事速度
描述：5000字仅推进1个情节点，节奏拖沓
严重度：中等
当前速度：慢
建议速度：中
建议：减少冗余描写，加快情节推进
```

### 示例2：节奏过快
```
【问题】第8章
类型：叙事速度
描述：2000字推进5个重要情节，匆忙感强
严重度：中等
当前速度：快
建议速度：中
建议：扩充关键情节，增加细节描写
```

### 示例3：描写比例失衡
```
【问题】第12章
类型：描写比例
描述：心理描写占比60%，动作描写仅10%
严重度：轻微
当前比例：心理60% 动作10% 对话20% 环境10%
建议比例：心理20% 动作30% 对话30% 环境10% 说明10%
建议：减少心理描写，增加动作和对话
```

### 示例4：场景过长
```
【问题】第15章战斗场景
类型：场景长度
描述：战斗场景长达8000字
严重度：中等
建议长度：1500-3000字
建议：拆分为多个小战斗，或穿插其他内容
```

### 示例5：段落节奏单调
```
【问题】第6章
类型：段落节奏
描述：90%的段落长度在100-150字之间
严重度：轻微
建议：长短段落交替，形成节奏感
示例：在长段落后插入短段落，制造停顿感
```

```yaml
  # 是否生成节奏曲线图

```

## 使用示例

### 基本使用
```bash
# 检测特定章节的节奏
forgeai review chapter 5 --agent pacing-checker

# 分析整个项目的节奏
forgeai review all --agent pacing-checker --analyze

# 生成节奏曲线图
forgeai review all --agent pacing-checker --plot
```

### 高级选项
```bash
# 只检测叙事速度

# 分析描写比例
```

## 常见问题

### Q: 如何判断节奏是否合适？
A: 节奏的合适性取决于场景类型和情节重要性。系统会根据场景类型（战斗、对话、日常等）和情节重要性（高潮、铺垫、过渡等）来判断节奏是否匹配。

### Q: 快节奏和慢节奏哪个更好？
A: 没有绝对的好坏，关键在于匹配场景需求：
- 高潮、战斗、危机 → 快节奏
- 情感、氛围、日常 → 慢节奏
- 成长、探索、铺垫 → 中节奏

### Q: 如何调整节奏？
A: 节奏调整方法：
- 加快节奏：减少描写，增加动作，缩短段落，跳跃时间
- 放慢节奏：增加细节，深化情感，延长段落，展开过程

### Q: 信息密度多少合适？
A: 建议每1000字推进1-2个情节点。密度过低会拖沓，密度过高会匆忙。具体还需根据场景类型调整。

## 最佳实践

1. **场景匹配节奏**：根据场景类型选择合适的节奏
2. **张弛有度**：快慢节奏交替，避免单调
3. **均衡描写**：保持各类描写的合理比例
4. **段落变化**：长短段落交替，形成节奏感
5. **控制场景长度**：避免单个场景过长或过短
6. **定期检查**：每完成一章，检查节奏是否合理
