# 时间线管理 Agent (timeline-agent)

## 概述

时间线管理Agent负责构建和维护故事的时间线，追踪事件发生的时间、角色的时间位置、时间跳跃等，确保时间逻辑的一致性。

## 核心功能

### 1. 时间线构建
自动从文本中提取时间信息，构建时间线：

```
时间线示例:
2024-01-01 (第1章): 李明到北京
2024-01-02 (第2章): 李明开始工作
2024-01-05 (第3章): 李明遇到王芳
2024-01-10 (第4章): 李明和王芳成为朋友
...
```

### 2. 时间标记识别
识别各种时间表达方式：

```
绝对时间:
- "2024年1月1日"
- "正月十五"
- "星期一"

相对时间:
- "第二天"
- "三天后"
- "一周以后"

模糊时间:
- "不久之后"
- "过了很久"
- "转眼间"
```

### 3. 时间跳跃检测
检测时间跳跃并验证合理性：

```
时间跳跃类型:
- 短跳跃：1-7天
- 中跳跃：1周-1月
- 长跳跃：1月以上

问题检测:
第5章: 2024-01-01
第6章: 2024-03-01 (跳跃2个月，无说明)
→ 警告：时间跳跃过大，建议添加说明
```

### 4. 多线时间追踪
追踪多条故事线的时间：

```
多线叙事示例:
A线 (主角):
- 第1章: 2024-01-01
- 第3章: 2024-01-02
- 第5章: 2024-01-03

B线 (配角):
- 第2章: 2024-01-01
- 第4章: 2024-01-02
- 第6章: 2024-01-03

交汇检测:
第7章: A线和B线交汇，时间是否一致？
```

### 5. 角色时间位置
追踪每个角色在时间线上的位置：

```
角色时间位置:
李明:
- 2024-01-01: 在北京
- 2024-01-05: 在上海
- 2024-01-10: 回到北京

王芳:
- 2024-01-01: 在上海
- 2024-01-05: 在上海
- 2024-01-08: 去北京

问题检测:
第6章: 李明和王芳在北京见面
但时间线显示：李明在上海，王芳在北京
→ 警告：角色位置冲突
```

### 6. 时间逻辑验证
验证时间相关的逻辑：

```
逻辑验证:
- 事件顺序：原因必须在结果之前
- 年龄增长：角色年龄必须随时间增长
- 季节变化：时间推进必须伴随季节变化
- 事件时长：事件时长不能超过时间跨度
```

## 使用示例

### 基本使用
```bash
# 构建时间线
forgeai timeline build

# 查看时间线
forgeai timeline view

# 检查时间逻辑
forgeai timeline check
```

### 高级选项
```bash
# 查看特定章节的时间信息
forgeai timeline view --chapter 5

# 查看特定角色的时间线
forgeai timeline view --character "李明"

# 导出时间线数据
forgeai timeline export --format json

# 生成时间线图表
forgeai timeline plot

# 检测时间跳跃
forgeai timeline check-jumps

# 验证多线时间
forgeai timeline verify-threads
```

## 配置选项

```yaml
timeline_agent:
  # 时间跳跃警告阈值
  jump_warning_threshold: 7天
  
  # 时间跳跃错误阈值
  jump_error_threshold: 30天
  
  # 是否检测模糊时间
  detect_fuzzy_time: true
  
  # 是否追踪角色位置
  track_character_location: true
  
  # 是否验证多线时间
  verify_threads: true
  
  # 时间格式
  date_format: "YYYY-MM-DD"
  
  # 是否自动构建时间线
  auto_build: true
  
  # 时间线数据存储路径
  timeline_path: ".forgeai/timeline.json"
```

## 常见问题

### Q: 如何处理模糊时间？
A: 系统会识别"不久之后""过了很久"等模糊表达，并尝试根据上下文推断具体时间。如果无法推断，会标记为"模糊时间"并在时间线中保留不确定性。

### Q: 如何处理回忆/倒叙？
A: 系统会识别回忆、倒叙等特殊叙事手法，并在时间线中标注。回忆内容会被放置在正确的时间位置，但不会影响主线时间推进。

### Q: 多线叙事的时间如何管理？
A: 系统会为每条故事线维护独立的时间线，并在线索交汇时验证时间的一致性。如果发现时间冲突，会发出警告。

### Q: 如何处理时间跳跃？
A: 系统会检测时间跳跃，并根据跳跃大小给出不同级别的提示：
- 短跳跃（1-7天）：正常，无需说明
- 中跳跃（1周-1月）：建议添加时间标记
- 长跳跃（1月以上）：强烈建议添加说明

## 最佳实践

1. **明确时间标记**：在时间跳跃时使用明确的时间表达
2. **定期检查**：每完成几章，运行时间线检查
3. **维护角色时间**：追踪角色在不同时间的位置和状态
4. **验证多线时间**：多线叙事时，确保线索交汇时时间一致
5. **使用时间线可视化**：生成时间线图表，直观查看时间分布
