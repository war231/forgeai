# 追读力检查 Agent (reader-pull-checker)

## 概述

追读力检查Agent评估文本吸引读者持续阅读的能力，这是网文创作的核心指标。它从读者心理学角度分析悬念设置、钩子强度、节奏把控等关键因素。

## 审查维度

### 1. 章节钩子
检测章节结尾是否设置有效钩子：

```
❌ 弱钩子示例:
"李明回到家，洗了个澡，就睡了。" (平淡无奇)

✅ 强钩子示例:
"李明刚要关门，突然发现门把手上挂着一个陌生的黑色信封..."
```

### 2. 悬念密度
检测悬念的分布和强度：

```
问题示例:
第1-5章：悬念密集，每章都有新谜团
第6-10章：悬念稀疏，全是日常描写 (追读力下降)

建议：保持悬念的节奏感，避免长时间无新悬念
```

### 3. 信息释放节奏
检测信息释放是否合理：

```
❌ 问题示例:
一次性揭露所有真相，读者失去探索乐趣

✅ 正确示例:
逐步释放线索，每章揭示一点，保持读者好奇心
```

### 4. 冲突强度
检测冲突的持续性和升级：

```
追读力公式:
冲突强度 × 不确定性 × 情感投入 = 追读力

问题示例:
连续3章无任何冲突，读者容易弃书
```

### 5. 情感钩子
检测情感共鸣点的设置：

```
✅ 情感钩子示例:
- 角色困境引发同情
- 不公待遇引发愤怒
- 浪漫互动引发期待
- 成长突破引发满足
```

### 6. 预期管理
检测读者预期的建立和满足：

```
预期建立 → 预期延迟 → 预期满足/反转

问题示例:
预期建立后立即满足，缺乏延迟过程 (追读力弱)
预期延迟过久不满足，读者失去耐心 (弃书风险)
```

## 评估指标

### 章节追读力评分
```
评分维度:
- 钩子强度 (0-10)
- 悬念密度 (0-10)
- 冲突强度 (0-10)
- 情感投入 (0-10)
- 节奏把控 (0-10)

综合评分 = 加权平均
```

### 弃书风险预测
```
风险因素:
- 连续N章无钩子
- 悬念长时间未推进
- 冲突强度持续下降
- 信息密度过低
- 节奏过于拖沓

风险等级: 低/中/高/极高
```

## 问题示例

### 示例1：弱章节结尾
```
【问题】第5章结尾
类型：章节钩子
描述：结尾平淡，无悬念或冲突
严重度：严重
当前钩子强度：2/10
建议：添加悬念元素，如意外发现、突发状况、关键线索等
示例：在结尾添加"就在这时，他注意到..."
```

### 示例2：悬念断层
```
【问题】第8-12章
类型：悬念密度
描述：连续5章无新悬念产生
严重度：严重
弃书风险：高
建议：在第8章引入新谜团，或在现有悬念上添加转折
```

### 示例3：信息一次性释放
```
【问题】第15章
类型：信息释放节奏
描述：一次性揭露3个重要真相，无悬念留存
严重度：中等
建议：分散到多章释放，每章揭露一个，保持探索感
```

### 示例4：冲突缺失
```
【问题】第6-9章
类型：冲突强度
描述：连续4章无任何冲突，纯日常描写
严重度：严重
弃书风险：极高
建议：引入人际冲突、外部压力、内心挣扎等冲突元素
```

## 配置选项

```yaml
reader_pull_checker:
  # 钩子强度阈值 (低于此值会警告)
  hook_threshold: 5
  
  # 悬念密度阈值 (每N章至少有1个新悬念)
  suspense_density: 3
  
  # 冲突缺失容忍度 (连续N章无冲突会警告)
  conflict_tolerance: 2
  
  # 信息释放建议 (重要真相建议分散到N章)
  info_release_span: 3
  
  # 弃书风险预测模型
  abandonment_model: conservative
  
  # 情感钩子检测
  check_emotional_hooks: true
  
  # 预期管理检测
  check_expectation_management: true
```

## 使用示例

### 基本使用
```bash
# 检测特定章节的追读力
forgeai review chapter 5 --agent reader-pull-checker

# 获取追读力评分
forgeai review chapter 5 --agent reader-pull-checker --score

# 预测弃书风险
forgeai review all --agent reader-pull-checker --predict-abandonment
```

### 高级选项
```bash
# 只检测章节钩子
forgeai review chapter 5 --agent reader-pull-checker --focus hooks

# 分析悬念分布
forgeai review all --agent reader-pull-checker --analyze-suspense

# 生成追读力曲线图
forgeai review all --agent reader-pull-checker --plot

# 对比优秀作品
forgeai review chapter 5 --agent reader-pull-checker --benchmark
```

## 常见问题

### Q: 什么是"钩子"？
A: 钩子是能够吸引读者继续阅读的悬念元素，通常设置在章节结尾。常见钩子类型：
- 悬念钩子：留下未解之谜
- 冲突钩子：危机即将爆发
- 转折钩子：意外情况发生
- 情感钩子：情感高潮或低谷

### Q: 悬念密度多少合适？
A: 根据网文创作经验，建议每2-3章引入新悬念或推进现有悬念。过密会导致信息过载，过疏会导致追读力下降。

### Q: 如何平衡信息释放和悬念保持？
A: 建议采用"洋葱剥皮"策略：每章揭示一层，同时引入新谜团。保持"已知/未知"的动态平衡。

### Q: 冲突强度如何量化？
A: 系统会分析冲突的类型、规模、紧迫性、不确定性等维度，综合计算冲突强度。内部冲突（心理挣扎）和外部冲突（人际对抗）都会被考虑。

## 最佳实践

1. **每章必设钩子**：养成在章节结尾设置悬念的习惯
2. **悬念节奏化**：大悬念推进时，穿插小悬念保持新鲜感
3. **冲突持续升级**：避免冲突强度持续下降，定期引入新冲突或升级现有冲突
4. **预期延迟满足**：建立预期后，延迟2-5章再满足，保持期待感
5. **情感共鸣点**：每3-5章设置一个情感共鸣场景
6. **定期自检**：每完成5章，运行追读力检查，识别问题区域
