# OOC检测 Agent (ooc-checker)

## 概述

OOC检测Agent负责识别角色行为与设定不符的情况，防止角色"崩人设"。这是小说创作中最常见的问题之一，直接影响读者对角色的认同感。

## 审查维度

### 1. 性格一致性
检测角色行为是否符合其核心性格特征：

```
角色设定: "李明是一个谨慎、多疑的人"

❌ OOC示例:
"李明毫不犹豫地打开了陌生邮件的附件"

✅ 符合设定:
"李明盯着那封邮件看了半天，最终还是没敢点开附件"
```

### 2. 能力边界
检测角色是否表现出超出其能力范围的行为：

```
角色设定: "张华是个普通高中生，没有任何武术基础"

❌ OOC示例:
"张华一个飞踢将持刀歹徒踢飞了两米远"

✅ 符合设定:
"张华吓得双腿发软，只能本能地往后退"
```

### 3. 知识范围
检测角色是否表现出不该拥有的知识：

```
角色设定: "王芳生活在古代，从未接触过现代科技"

❌ OOC示例:
"王芳心想这东西要是能通电就好了"

✅ 符合设定:
"王芳完全不明白这个铁盒子怎么会发出人声"
```

### 4. 情感逻辑
检测角色情感反应是否符合其性格和处境：

```
角色设定: "赵刚是个极度冷静的人，很少表露情感"

❌ OOC示例:
"赵刚激动得热泪盈眶，大声喊叫起来"

✅ 符合设定:
"赵刚的嘴角微微上扬，这是他难得的笑容"
```

### 5. 说话方式
检测角色对话是否符合其语言习惯：

```
角色设定: "老李是个粗人，说话直来直去"

❌ OOC示例:
"老李委婉地表示：'这个嘛，可能需要再考虑考虑'"

✅ 符合设定:
"老李直接说：'不行，这事儿没门'"
```

## 检测机制

### 静态分析
- 对比角色行为与设定文件
- 检查能力值是否超出上限
- 验证知识背景是否合理

### 动态追踪
- 追踪角色成长轨迹
- 识别渐进式性格变化
- 区分"成长"与"OOC"

### 上下文关联
- 考虑特殊情境下的行为变化
- 识别情绪波动对行为的影响
- 处理角色伪装/演戏的情况

## 问题示例

### 示例1：能力越界
```
【问题】第5章第3段
角色：林小雨
设定：普通大学生，无特殊技能
问题：单手举起50斤重物
严重度：中等
建议：改为两人一起搬，或使用工具
```

### 示例2：性格突变
```
【问题】第12章第8段
角色：陈刚
设定：极度谨慎，从不冒险
问题：未经思考就答应陌生人的冒险提议
严重度：严重
建议：增加内心挣扎，或设置特殊动机
```

### 示例3：知识穿越
```
【问题】第7章第2段
角色：古代书生
设定：明朝人，无现代知识
问题：使用"心理暗示"等现代概念
严重度：轻微
建议：改为古代用语，如"攻心之术"
```

## 配置选项

```yaml
ooc_checker:
  # 检测敏感度 (low/medium/high)
  sensitivity: medium
  
  # 能力容忍度 (允许超出设定值的百分比)
  capability_tolerance: 10%
  
  # 是否检测说话方式
  check_speech_pattern: true
  
  # 是否考虑角色成长
  allow_character_growth: true
  
  # 成长判定阈值 (多少章后开始考虑成长)
  growth_threshold: 10
  
  # 忽略的OOC类型
  ignore_types:
    - "梦境"
    - "幻觉"
    - "角色扮演"
```

## 使用示例

### 基本使用
```bash
# 检测特定章节
forgeai review chapter 5 --agent ooc-checker

# 检测整个项目
forgeai review all --agent ooc-checker
```

### 高级选项
```bash
# 只检测特定角色
forgeai review chapter 5 --agent ooc-checker --character "李明"

# 设置敏感度
forgeai review chapter 5 --agent ooc-checker --sensitivity high

# 输出详细报告
forgeai review chapter 5 --agent ooc-checker --verbose
```

## 常见问题

### Q: 角色成长会被误判为OOC吗？
A: 不会。OOC检测会追踪角色成长轨迹，区分合理的性格演变和突兀的行为改变。可以通过 `allow_character_growth` 和 `growth_threshold` 配置来调整判定标准。

### Q: 特殊情境下的行为变化如何处理？
A: OOC检测会考虑情境因素。例如，一个冷静的人在生死关头可能会失控，这是合理的情感反应。建议在设定中标注角色的"极端反应模式"。

### Q: 如何处理角色伪装/演戏？
A: 在配置中添加忽略类型，或在文本中明确标注"假装""演戏"等关键词。OOC检测会自动识别这些特殊情况。

## 最佳实践

1. **完善角色设定**：在开始创作前，尽可能详细地定义角色性格、能力、知识背景
2. **标注成长节点**：在角色成长的关键章节添加注释，帮助系统识别
3. **使用角色模板**：创建角色模板文件，确保设定的一致性
4. **定期检查**：每完成一章就运行OOC检测，避免问题累积
5. **人工复核**：对于严重度"轻微"的问题，可以人工判断是否需要修改

## 相关Agent
