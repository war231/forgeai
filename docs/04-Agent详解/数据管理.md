# 数据管理 Agent (data-agent)

## 概述

数据管理Agent负责ForgeAI项目中所有数据的存储、读取、备份和迁移，包括项目配置、角色设定、时间线数据、上下文数据、审查结果等。

## 核心功能

### 1. 项目数据管理
管理项目的基本配置和数据：

```
项目数据结构:
.forgeai/
├── config.yaml          # 项目配置
├── characters/          # 角色设定
│   ├── 李明.yaml
│   └── 王芳.yaml
├── settings/            # 世界设定
│   ├── 世界观.yaml

├── context.json         # 上下文数据

├── plots/               # 情节图谱
│   └── main-plot.json
└── backups/             # 备份数据
    └── 2024-01-15/
```

### 6. 数据备份与恢复
管理数据的备份和恢复：

```
备份策略:
- 自动备份：每次重要操作后自动备份
- 定期备份：每天/每周定期备份
- 手动备份：用户主动触发备份

备份内容:
- 项目配置
- 角色设定
- 时间线数据
- 上下文数据
- 审查结果
- 创作内容

恢复选项:
- 完全恢复：恢复所有数据
- 部分恢复：只恢复特定数据
- 增量恢复：恢复到特定时间点
```

### 7. 数据迁移
管理数据的导入导出和迁移：

```
导出格式:
- JSON：通用格式，便于其他工具处理
- YAML：人类可读，便于编辑
- CSV：表格数据，便于分析
- Markdown：文档格式，便于阅读

导入来源:
- 其他创作工具（如Scrivener）
- 手动编写的设定文件
- 其他ForgeAI项目

迁移场景:
- 项目升级：数据格式升级
- 项目合并：多个项目合并
- 项目拆分：项目拆分为多个子项目
```

## 使用示例

### 基本使用
```bash
# 查看项目数据
forgeai data view

# 导出项目数据
forgeai data export --format json --output backup.json

# 导入项目数据
forgeai data import --input backup.json

# 备份项目数据
forgeai data backup

# 恢复项目数据
forgeai data restore --backup 2024-01-15
```

### 高级选项
```bash
# 只导出角色数据
forgeai data export --type characters --output characters.json

# 只导出特定角色的数据
forgeai data export --character "李明" --output liming.yaml

# 导出审查结果
forgeai data export --type reviews --output reviews.json

# 查看数据统计
forgeai data stats

# 验证数据完整性
forgeai data validate

# 清理无效数据
forgeai data cleanup

# 迁移数据格式
forgeai data migrate --from 1.0 --to 2.0
```

## 配置选项

```yaml
data_agent:
  # 数据存储路径
  data_path: ".forgeai"
  
  # 自动备份
  auto_backup: true
  
  # 备份间隔（天）
  backup_interval: 1
  
  # 备份保留数量
  backup_retention: 10
  
  # 数据格式版本
  data_version: "1.0"
  
  # 导出格式
  export_format: "json"
  
  # 是否压缩备份
  compress_backup: true
  
  # 数据验证
  validate_on_save: true
  
  # 数据迁移
  auto_migrate: true
```

## 常见问题

### Q: 数据存储在哪里？
A: 默认存储在项目根目录的 `.forgeai/` 文件夹中。可以在配置中修改存储路径。

### Q: 如何备份到云端？
A: ForgeAI支持将数据导出为JSON/YAML格式，可以上传到任何云存储服务。未来版本将支持直接集成云存储。

### Q: 数据格式升级会丢失数据吗？
A: 不会。数据迁移会保留所有数据，只是转换格式。迁移前会自动备份，确保数据安全。

### Q: 如何从其他工具导入数据？
A: 目前支持导入JSON和YAML格式的数据。对于Scrivener等工具，需要先导出为通用格式，再导入ForgeAI。

### Q: 数据损坏如何恢复？
A: 如果数据损坏，可以从最近的备份恢复。建议启用自动备份，定期备份重要数据。

## 最佳实践

1. **定期备份**：启用自动备份，或定期手动备份
2. **验证数据**：定期运行数据验证，确保数据完整性
3. **版本控制**：将 `.forgeai/` 目录纳入Git版本控制
4. **导出备份**：重要节点导出数据，存储到安全位置
5. **数据迁移**：升级前备份数据，迁移后验证数据
