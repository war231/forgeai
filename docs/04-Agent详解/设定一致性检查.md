# 设定一致性检查

> **consistency-checker Agent** - 确保章节内容与已有设定保持一致

---

## Agent 职责

设定一致性检查Agent负责检查章节内容是否与以下设定保持一致：

1. **世界观设定** - 世界规则、地理设定、势力分布
2. **角色设定** - 角色能力、性格、外貌、背景
3. **金手指设定** - 金手指能力、使用限制、成长路径
4. **力量体系** - 修炼等级、战斗规则、技能设定

---

## 审查维度

### 1. 世界观一致性

**检查项:**
- 地理设定是否正确
- 势力关系是否合理
- 世界规则是否遵守
- 特殊设定是否一致

**问题示例:**

❌ **错误示例:**
```
设定: 青云宗位于东大陆
章节: 主角来到青云宗，发现这里属于西大陆...
```

✅ **正确示例:**
```
设定: 青云宗位于东大陆
章节: 主角穿越东大陆，终于来到青云宗山门...
```

---

### 2. 角色设定一致性

**检查项:**
- 角色能力是否超出设定
- 角色性格是否符合设定
- 角色外貌描述是否一致
- 角色背景是否正确引用

**问题示例:**

❌ **错误示例:**
```
设定: 李天是筑基期修士
章节: 李天施展元婴期的神通，一掌拍碎了山峰...
```

✅ **正确示例:**
```
设定: 李天是筑基期修士
章节: 李天运转筑基期的灵力，施展了一记烈焰掌...
```

---

### 3. 金手指一致性

**检查项:**
- 金手指能力是否正确
- 使用限制是否遵守
- 成长进度是否合理
- 副作用是否体现

**问题示例:**

❌ **错误示例:**
```
设定: 金手指每天只能使用一次
章节: 李天连续使用了三次金手指...
```

✅ **正确示例:**
```
设定: 金手指每天只能使用一次
章节: 李天犹豫了一下，今天已经用过金手指了，只能另想办法...
```

---

### 4. 力量体系一致性

**检查项:**
- 修炼等级是否正确
- 战斗规则是否合理
- 技能效果是否一致
- 境界差距是否合理

**问题示例:**

❌ **错误示例:**
```
设定: 筑基期修士无法飞行
章节: 李天虽然只是筑基期，却能在空中自由飞行...
```

✅ **正确示例:**
```
设定: 筑基期修士无法飞行
章节: 李天踩着飞剑，勉强在空中滑翔了一段距离...
```

## 审查流程

### Step 1: 加载设定

从项目设定文件加载：

```
2-设定/
├── 世界观设定.md
├── 角色设定.md
├── 金手指设定.md
└── 力量体系.md
```

### Step 2: 提取章节实体

识别章节中的关键实体：

- 人物名称
- 地点名称
- 物品名称
- 技能名称

### Step 3: 一致性检查

对每个实体进行一致性检查：

1. 查找实体设定
2. 比对章节描述
3. 标记不一致之处
4. 生成修复建议

### Step 4: 生成报告

输出审查报告：

```markdown
# 设定一致性审查报告

## 审查结果

- 状态: ⚠️ 警告
- 评分: 7/10
- 问题数量: 3

## 发现的问题

### 问题1: 角色能力超出设定

**位置:** 第15段
**描述:** 李天使用了元婴期神通，但设定中他是筑基期
**严重程度:** 高
**修复建议:** 将神通改为筑基期可用的技能，如"烈焰掌"

### 问题2: 地理设定错误

**位置:** 第8段
**描述:** 青云宗被描述为位于西大陆，但设定中位于东大陆
**严重程度:** 中
**修复建议:** 将"西大陆"改为"东大陆"

### 问题3: 金手指使用超限

**位置:** 第20段
**描述:** 金手指连续使用3次，但设定中每天只能使用1次
**严重程度:** 高
**修复建议:** 删除多余的金手指使用，或添加限制解除的情节
```

## 配置选项

### 基础配置

```json
{
  "agents": {
    "consistency-checker": {
      "enabled": true,
      "strictness": "high"
    }
  }
}
```

### 详细配置

```json
{
  "agents": {
    "consistency-checker": {
      "enabled": true,
      "strictness": "high",
      "check_worldview": true,
      "check_characters": true,
      "check_golden_finger": true,
      "check_power_system": true,
      "ignore_minor_errors": false,
      "max_issues": 10
    }
  }
}
```

**配置说明:**

| 选项 | 说明 | 默认值 |
|------|------|--------|
| `check_worldview` | 检查世界观一致性 | true |
| `check_characters` | 检查角色设定一致性 | true |
| `check_golden_finger` | 检查金手指一致性 | true |
| `check_power_system` | 检查力量体系一致性 | true |
| `ignore_minor_errors` | 忽略轻微错误 | false |
| `max_issues` | 最大问题数量 | 10 |

## 使用示例

### 基础使用

```bash
# 审查章节
forgeai check review 10 --agents consistency
```

### 高级使用

```bash
# 只检查角色设定
forgeai check review 10 --agents consistency --check characters

# 忽略轻微错误
forgeai check review 10 --agents consistency --ignore-minor
```

## 常见问题

### Q1: 如何添加新设定？

**A:** 在 `2-设定/` 目录添加设定文件：

```markdown
# 角色设定

## 李天

- **境界**: 筑基期
- **性格**: 坚毅、果断
- **金手指**: 每日一次的时间回溯
- **外貌**: 黑衣少年，剑眉星目
```

### Q2: 设定冲突怎么办？

**A:** 设定优先级：

1. 章节内设定 > 项目设定
2. 新设定 > 旧设定
3. 明确设定 > 隐含设定

建议：统一更新设定文件，避免冲突。

### Q3: 如何处理合理突破？

**A:** 标记为"合理突破"：

```markdown
<!-- consistency: allow-break -->
李天在生死关头，强行催动金手指，突破了每日一次的限制...
```

## 最佳实践

### 1. 及时更新设定

每次创作后更新设定：

```bash
# 添加新角色
forgeai entity add "王长老" --type character --tier important

# 更新角色状态
forgeai entity update "李天" --field "境界" --value "金丹期"
```

### 2. 定期检查设定

定期检查设定一致性：

```bash
# 检查所有章节
for i in {1..20}; do
  forgeai check review $i --agents consistency
done
```

- 出身:
- 经历:
