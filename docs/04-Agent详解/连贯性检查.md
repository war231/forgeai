# 连贯性检查 Agent (continuity-checker)

## 概述

连贯性检查Agent负责检测叙事逻辑的断裂和前后矛盾，确保故事发展的合理性和流畅性。这是维持读者沉浸感的关键Agent。

## 审查维度

### 1. 时间连贯性
检测时间线是否合理：

```
❌ 问题示例:
"他昨天刚到北京" (第5章)
"他已经在这住了三个月" (第6章，时间跨度仅1天)

✅ 正确示例:
"他昨天刚到北京" (第5章)
"今天是他到北京的第二天" (第6章)
```

### 2. 空间连贯性
检测地点转换是否合理：

```
❌ 问题示例:
"李明在上海的办公室里" (第3章结尾)
"李明走出北京的家门" (第4章开头，无过渡)

✅ 正确示例:
"李明在上海的办公室里" (第3章结尾)
"连夜坐飞机赶回北京，李明推开家门" (第4章开头)
```

### 3. 因果连贯性
检测事件因果关系是否合理：

```
❌ 问题示例:
"公司突然破产了" (无任何征兆和原因)

✅ 正确示例:
"连续三个季度亏损，资金链断裂，公司最终宣布破产"
```

### 4. 情节连贯性
检测情节发展是否跳跃：

```
❌ 问题示例:
第5章：主角刚认识反派
第6章：主角已经完全了解反派的阴谋 (缺少探索过程)

✅ 正确示例:
第5章：主角刚认识反派
第6章：主角开始调查，发现疑点
第7章：主角逐步揭开真相
```

### 5. 情绪连贯性
检测角色情绪变化是否合理：

```
❌ 问题示例:
"李明悲痛欲绝" (段落结尾)
"李明兴高采烈地跑出去" (下一段开头)

✅ 正确示例:
"李明悲痛欲绝"
"经过一夜的消化，李明决定振作起来"
```

## 检测机制

### 章节间检测
- 对比相邻章节的结尾和开头
- 检查时间、地点、状态的衔接
- 识别缺失的过渡段落

### 跨章节检测
- 追踪长线情节的发展
- 检查伏笔和回收的一致性
- 验证设定的前后呼应

### 逻辑链检测
- 构建事件因果关系图
- 识别断裂的因果链
- 检测循环依赖和矛盾

## 问题示例

### 示例1：时间跳跃
```
【问题】第5章 → 第6章
类型：时间连贯性
描述：第5章结尾是周一早上，第6章开头直接跳到周五
严重度：中等
建议：添加时间过渡说明，或调整章节划分
```

### 示例2：空间跳跃
```
【问题】第8章第3段 → 第4段
类型：空间连贯性
描述：角色从公司直接出现在家里，无移动过程
严重度：轻微
建议：添加"下班回到家"等过渡语句
```

### 示例3：因果缺失
```
【问题】第12章第5段
类型：因果连贯性
描述：角色突然决定辞职，无任何铺垫
严重度：严重
建议：前文添加工作压力、矛盾冲突等铺垫
```

### 示例4：情节跳跃
```
【问题】第15章 → 第16章
类型：情节连贯性
描述：调查过程缺失，直接得到结论
严重度：严重
建议：增加调查章节，或使用回忆补叙
```

## 配置选项

```yaml
continuity_checker:
  # 时间跳跃容忍度 (章节数)
  time_jump_tolerance: 1
  
  # 是否检测空间跳跃
  check_spatial_jump: true
  
  # 因果链检测深度
  causal_chain_depth: 3
  
  # 情节跳跃检测阈值
  plot_jump_threshold: medium
  
  # 情绪变化检测
  check_emotion_continuity: true
  
  # 情绪变化容忍度 (段落间隔)
  emotion_tolerance: 2
  
  # 忽略的场景类型
  ignore_scenes:
    - "梦境"
    - "回忆"
    - "想象"
```

## 使用示例

### 基本使用
```bash
# 检测特定章节的连贯性
forgeai review chapter 5 --agent continuity-checker

# 检测章节间的衔接
forgeai review chapters 5-6 --agent continuity-checker --focus transition

# 检测整个项目的连贯性
forgeai review all --agent continuity-checker
```

### 高级选项
```bash
# 只检测时间连贯性
forgeai review chapter 5 --agent continuity-checker --type temporal

# 只检测因果连贯性
forgeai review chapter 5 --agent continuity-checker --type causal

# 设置检测深度
forgeai review chapter 5 --agent continuity-checker --depth 5

# 输出修复建议
forgeai review chapter 5 --agent continuity-checker --suggest
```

## 常见问题

### Q: 如何处理时间跳跃？
A: 合理的时间跳跃需要明确标注。例如："三天后"、"转眼到了年底"。系统会识别这些时间标记，并验证后续内容的一致性。

### Q: 回忆/倒叙会被误判吗？
A: 不会。系统会识别回忆、倒叙等特殊叙事手法，并跳过这些场景的连贯性检测。可以在配置中添加 `ignore_scenes`。

### Q: 多线叙事如何处理？
A: 系统会分别追踪每条故事线，检测各条线内部的连贯性，以及线索交汇时的逻辑一致性。

### Q: 如何处理省略的过渡？
A: 对于轻微的空间跳跃（如"下班回家"），系统会给出"轻微"级别的建议。作者可以选择补充过渡，或保持简洁叙事。

## 最佳实践

1. **使用时间标记**：在时间跳跃时明确标注，如"第二天""一周后"
2. **添加过渡段落**：在重大场景转换时，添加简短的过渡说明
3. **铺垫因果关系**：重要事件前添加征兆、暗示、动机铺垫
4. **检查章节衔接**：每完成一章，检查与前后章节的衔接
5. **维护情节图谱**：使用 ForgeAI 的情节追踪功能，可视化检查情节发展
