# write 命令详解

## 概述

`write` 命令是 ForgeAI 的**一键写作**命令,自动完成写前准备和提示词生成,让你专注于创作本身。

## 功能

执行 `write` 命令时,系统会自动:

1. **写前检查** - 检查过期伏笔、叙事债务、节奏平衡等
2. **提取上下文** - 提取前文章节、活跃实体、伏笔、RAG检索结果
3. **生成提示词** - 组装完整的创作提示词,可直接发送给AI

## 使用方式

### 基础用法

```bash
# 准备第20章的创作
forgeai write 20
```

### 指定主题

```bash
# 重点写主角和女主的感情线
forgeai write 20 --query "主角和女主角的感情发展"

# 重点写战斗场面
forgeai write 25 --query "主角与反派的终极对决"

# 重点写世界观揭示
forgeai write 30 --query "揭示修炼体系的真相"
```

### 智能上下文模式

```bash
# 使用智能上下文(按优先级召回,控制字数)
forgeai write 20 --smart --max-chars 10000

# 智能模式 + 主题指导
forgeai write 20 --smart --query "感情线" --max-chars 8000
```

### 保存到文件

```bash
# 将提示词保存到文件
forgeai write 20 --output prompt_20.txt
```

## 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `chapter` | 章节号(必填) | - |
| `--query` | 主题指导,用于RAG检索和写作重点 | 无 |
| `--smart` | 使用智能上下文模式 | 否 |
| `--max-chars` | 最大上下文字数(仅--smart时生效) | 8000 |
| `--output` | 输出文件路径 | 终端输出 |
| `--project-root` | 项目根目录 | 自动查找 |

## 工作流程

```
用户执行: forgeai write 20 --query "感情线"
    ↓
Step 1: 写前检查
    ├─ 检查过期伏笔
    ├─ 检查叙事债务
    ├─ 检查节奏平衡
    └─ 生成提醒列表
    ↓
Step 2: 提取上下文
    ├─ 前5章正文(第15-19章)
    ├─ 活跃实体(第10-19章)
    ├─ 待回收伏笔
    ├─ RAG检索("感情"相关片段)
    └─ 组装上下文
    ↓
Step 3: 生成提示词
    ├─ 组装写前检查结果
    ├─ 组装上下文信息
    ├─ 添加主题指导
    ├─ 添加写作要求
    └─ 输出完整提示词
    ↓
用户将提示词发送给AI
    ↓
AI创作第20章正文
```

## 与其他命令的关系

### write vs context

- `context` - 仅提取上下文,不包含写前检查
- `write` - 完整流程(写前检查 + 上下文 + 提示词)

### write vs check before

- `check before` - 仅执行写前检查
- `write` - 包含写前检查 + 更多准备步骤

### 推荐工作流

```bash
# 方式1: 一键准备(推荐)
forgeai write 20 --query "感情线" > prompt.txt
# 将prompt.txt发送给AI

# 方式2: 分步执行(需要精细控制时)
forgeai check before 20          # 先检查问题
forgeai context 20 --smart       # 再提取上下文
# 手动组装提示词

# 方式3: 写后自动化
forgeai write 20 --output prompt.txt
# AI创作完成后
forgeai check after 20 4-正文/第20章.md  # 自动索引/评分/更新
```

## 常见问题

### Q: 提示词太长怎么办?

A: 使用 `--smart` 模式并设置 `--max-chars`:

```bash
forgeai write 20 --smart --max-chars 6000
```

### Q: 如何让AI重点写某个方面?

A: 使用 `--query` 参数:

```bash
forgeai write 20 --query "主角突破境界"
```

### Q: 写前检查发现问题怎么办?

A: 系统会在提示词中标注问题,AI创作时会自动注意。你也可以先手动处理问题:

```bash
# 查看详细问题
forgeai check before 20

# 解决问题后重新准备
forgeai write 20
```

### Q: 可以保存提示词供以后使用吗?

A: 可以,使用 `--output` 参数:

```bash
forgeai write 20 --output prompts/chapter_20.txt
```

## 最佳实践

1. **每次写作前都使用** - 确保上下文完整,避免矛盾
2. **善用主题指导** - 让AI聚焦在你想要的情节上
3. **关注写前提醒** - 及时处理过期伏笔和节奏问题
4. **保存提示词** - 方便后续调整和复用
5. **结合写后流水线** - 写作完成后自动索引和更新
