# 写后处理流程详解

## 概述

写完一章后,需要执行**写后流水线**(post-write pipeline),自动完成索引、实体提取、评分、进度更新等工作,确保新章节被系统正确记录和分析。

## 为什么需要写后处理?

写后处理是 ForgeAI 的核心机制,它确保:

1. **RAG检索可用** - 新章节被索引后,后续章节可以检索到相关内容
2. **实体状态同步** - 人物、地点等实体的状态自动更新
3. **质量监控** - AI味评分帮助保持文风自然
4. **进度追踪** - 系统知道当前写到哪儿了
5. **提前预警** - 为下一章预检,提前发现问题

## 使用方式

### 基础用法

```bash
# 写完第20章后,执行写后处理
forgeai check after 20 4-正文/第20章.md

# 或使用旧命令(兼容)
forgeai post-write 20 4-正文/第20章.md
```

### 跳过AI味评分

```bash
# 如果不需要评分,可以跳过(更快)
forgeai check after 20 4-正文/第20章.md --no-score
```

### 使用LLM提取实体

```bash
# 使用LLM进行更精确的实体提取(需要API Key)
forgeai check after 20 4-正文/第20章.md --llm
```

## 处理步骤详解

### Step 1: 索引章节

**作用**: 将新章节分块并向量化,存入RAG检索库

**流程**:
```
章节文本
    ↓
分块(500字/块)
    ↓
向量化(Embedding模型)
    ↓
存入向量数据库
    ↓
后续章节可检索
```

**输出**: 索引的分块数量

**示例**: 第20章3245字 → 6个分块

---

### Step 2: 提取实体

**作用**: 识别章节中的人物、地点、物品等实体,并更新状态

**提取内容**:
- **人物** - 出场、状态变化、关系变化
- **地点** - 新地点、地点转换
- **物品** - 新物品、物品归属变化
- **势力** - 势力关系变化

**状态变更示例**:
```
李天: 金丹初期 → 金丹中期
林月: 被抓 → 获救
神秘玉佩: 未出现 → 出现
```

**输出**: 提取的实体数量和状态变更数量

---

### Step 3: AI味评分

**作用**: 检测AI写作痕迹,保持文风自然

**检测维度**:
- 句式重复度
- 过度修饰词
- 情感表达模式
- 对话模式
- 段落结构

**评分标准**:
- **> 0.7** - 优秀(自然流畅)
- **0.5-0.7** - 良好(基本自然)
- **< 0.5** - 需优化(有明显AI痕迹)

**输出**: 评分值和等级

---

### Step 4: 更新进度

**作用**: 记录章节完成,更新项目进度

**更新内容**:
- 当前章节号
- 总字数
- 创作阶段
- 完成时间

**输出**: 进度更新状态

---

### Step 5: 下一章预检

**作用**: 提前检查下一章可能遇到的问题

**检查项目**:
- 超期伏笔提醒
- 叙事债务提醒
- 节奏平衡提醒
- 阅读力趋势

**输出**: 提醒列表

---

## 完整工作流

### 推荐流程

```bash
# 1. 写前准备
forgeai write 20 --query "感情线" --output prompt_20.txt

# 2. AI创作(将prompt发送给AI)

# 3. 保存正文到文件
# (将AI生成的内容保存到 4-正文/第20章.md)

# 4. 写后处理
forgeai check after 20 4-正文/第20章.md

# 5. (可选)详细审查
forgeai check review 20

# 6. 继续下一章
forgeai write 21 --query "战斗场面"
```

### 批量处理

```bash
# 如果写了多章,可以批量处理
for i in {20..25}; do
    forgeai check after $i "4-正文/第${i}章.md"
done
```

## 常见问题

### Q: 写后处理需要多长时间?

A: 通常5-15秒,取决于章节长度和是否使用LLM提取实体。

### Q: 可以跳过某些步骤吗?

A: 可以,使用参数控制:
- `--no-score` - 跳过AI味评分
- 不使用 `--llm` - 使用规则提取实体(更快)

### Q: 如果处理失败怎么办?

A: 查看错误信息,常见问题:
- 文件路径错误 → 检查路径是否正确
- 数据库锁定 → 确保没有其他进程占用
- API错误 → 检查API Key和网络

### Q: 需要每次都执行写后处理吗?

A: **强烈建议每次都执行**。如果不执行:
- RAG检索不到新章节
- 实体状态不会更新
- 进度不会记录
- 下一章预检无法进行

### Q: 写后处理可以重复执行吗?

A: 可以。重复执行会覆盖之前的数据,不会产生重复记录。

## 与其他命令的关系

### write vs check after

- `write` - 写前准备(检查+上下文+提示词)
- `check after` - 写后处理(索引+实体+评分+更新)

### check after vs check review

- `check after` - 自动化处理(5步流水线)
- `check review` - 详细审查(多Agent并行审查)

**推荐顺序**: `check after` → `check review`

## 最佳实践

1. **每次写完立即执行** - 不要积压多章
2. **关注AI味评分** - 低于0.5时考虑优化
3. **查看下一章提醒** - 提前规划内容
4. **定期审查** - 每5-10章做一次详细审查
5. **保存处理报告** - 方便后续分析

## 自动化建议

### VSCode扩展

使用VSCode扩展可以一键完成:
- 写前准备
- AI创作
- 写后处理
- 审查

### 脚本自动化

创建脚本 `auto_chapter.sh`:

```bash
#!/bin/bash
CHAPTER=$1
QUERY=$2

# 写前准备
forgeai write $CHAPTER --query "$QUERY" --output prompt_${CHAPTER}.txt

echo "请将 prompt_${CHAPTER}.txt 发送给AI创作"
read -p "创作完成后按回车继续..."

# 写后处理
forgeai check after $CHAPTER "4-正文/第${CHAPTER}章.md"

echo "第${CHAPTER}章处理完成!"
```

使用:
```bash
./auto_chapter.sh 20 "主角和女主的感情发展"
```
