# 数据结构说明

本文档详细说明ForgeAI项目中使用的各种数据结构。

---

## 项目数据结构

### 项目根目录

```
我的小说/
├── .forgeai/              # ForgeAI配置和数据目录
│   ├── config.yaml        # 项目配置
│   ├── characters/        # 角色设定
│   │   ├── 李明.yaml
│   │   └── 王芳.yaml
│   ├── settings/          # 世界设定
│   │   ├── 世界观.yaml
│   │   └── 地点.yaml
│   ├── plots/             # 情节大纲
│   │   └── main-plot.yaml
│   ├── timeline.json      # 时间线数据
│   ├── context.json       # 上下文数据
│   ├── reviews/           # 审查结果
│   │   ├── chapter-001.json
│   │   └── chapter-002.json
│   ├── backups/           # 备份数据
│   │   └── 2024-01-15/
│   └── logs/              # 日志文件
│       └── forgeai.log
├── chapters/              # 章节内容
│   ├── chapter-001.md
│   ├── chapter-002.md
│   └── ...
├── output/                # 输出目录
└── README.md              # 项目说明
```

## 配置数据结构

### config.yaml

项目配置文件，YAML格式。

```yaml
api:
  provider: openai
  key: "your-api-key"
  model: "gpt-4"
  base_url: "https://api.openai.com/v1"

project:
  name: "我的小说"
  genre: "都市"
  target_words: 500000
  target_chapters: 200

writing:
  words_per_chapter: 2500
  style: "轻松幽默"
  perspective: "第三人称"

review:
  auto_review: true
  agents:
    - consistency-checker
    - ooc-checker
```

详细说明请参考 [配置文件](配置文件.md)。

## 角色数据结构

### 角色文件 (characters/*.yaml)

角色设定文件，YAML格式。

```yaml
# 基本信息
name: 李明
age: 25
gender: 男
occupation: 后端工程师
company: 星辰科技

# 外貌描写
appearance:
  height: 175cm
  weight: 65kg
  build: 偏瘦
  features:
    - 戴黑框眼镜
    - 黑色短发
    - 目光锐利
  clothing_style:
    - 简单休闲
    - T恤牛仔裤

# 性格特征
personality:
  core_traits:
    - 谨慎
    - 聪明
    - 内向
  secondary_traits:
    - 固执
    - 完美主义
  strengths:
    - 逻辑思维强
    - 观察力敏锐
  weaknesses:
    - 社交能力弱
    - 过度谨慎

# 背景故事
background: |
  李明出生于普通工薪家庭，
  从小对计算机感兴趣...

# 能力值 (1-10)
abilities:
  编程: 9
  逻辑思维: 8
  观察: 8
  社交: 3
  表达: 4

# 当前状态
current_state:
  location: 北京
  mood: 平静
  goals:
    - 完成当前项目
    - 提升团队管理能力

# 人际关系
relationships:
  王芳:
    type: 朋友
    closeness: 7
    description: 公司同事，为数不多的朋友
  张刚:
    type: 同事
    closeness: 5
    description: 同组工程师

# 成长轨迹
growth:
  - chapter: 20
    change: "开始带团队，被迫社交"
    abilities:
      社交: 3 → 4
  - chapter: 100
    change: "困境中学会寻求帮助"
    abilities:
      社交: 4 → 6
      表达: 4 → 7
```

## 设定数据结构

### 世界设定文件 (settings/*.yaml)

世界设定文件，YAML格式。

```yaml
# 基本信息
name: 现代都市
time_period: 2024年
technology_level: 现代
genre: 都市职场

# 世界规则
world_rules:
  physical:
    - "遵循现实物理法则"
    - "无超自然元素"
  social:
    - "遵循现实社会规则"
    - "法律制度与现实一致"
  technology:
    - "科技水平与现实一致"
    - "无未来科技"

# 地点设定
locations:
  北京:
    type: 城市
    role: 主要舞台
    climate: 温带季风
    population: 2100万
    districts:
      朝阳区:
        type: 商业区
        features:
          - CBD
          - 三里屯
      海淀区:
        type: 教育区
        features:
          - 中关村
          - 大学集中
    landmarks:
      - name: 天安门
        type: 地标
        significance: 政治中心

  公司:
    type: 建筑物
    location: 北京海淀区中关村
    floors: 20
    description: |
      一家中型互联网公司...
```

## 情节数据结构

### 情节大纲文件 (plots/*.yaml)

情节大纲文件，YAML格式。

```yaml
name: 主线情节
genre: 都市职场
theme: 成长与蜕变

# 三幕结构
structure:
  first_act:
    chapters: 1-50
    theme: "新人入职，适应环境"
    percentage: 25%
    setup:
      - chapter: 1
        event: "李明入职星辰科技"
        significance: "激励事件"
      - chapter: 10
        event: "遇到王芳，建立友谊"
      - chapter: 50
        event: "决定深入调查真相"
        significance: "第一幕高潮"

  second_act:
    chapters: 51-150
    theme: "调查真相，成长蜕变"
    percentage: 50%
    confrontation:
      - chapter: 100
        event: "陷入困境，几乎放弃"
        significance: "中点低谷"
      - chapter: 150
        event: "真相大白"
        significance: "第二幕高潮"

  third_act:
    chapters: 151-200
    theme: "解决问题，新的开始"
    percentage: 25%
    resolution:
      - chapter: 200
        event: "新的开始"
        significance: "结局"

# 支线情节
subplots:
  romance:
    name: 李明与王芳的情感线
    chapters: 10-190
    beats:
      - chapter: 10
        event: "相遇"
      - chapter: 190
        event: "表白在一起"

# 关键场景
key_scenes:
  climaxes:
    - chapter: 50
      type: 第一幕高潮
      intensity: 8/10
    - chapter: 150
      type: 第二幕高潮
      intensity: 9/10

  turning_points:
    - chapter: 30
      event: "发现异常数据"
      significance: "改变故事走向"
```

## 数据验证

ForgeAI会自动验证数据结构的完整性：

```bash
# 验证数据
forgeai data validate

# 验证特定数据
forgeai validate characters
forgeai validate settings
forgeai validate plots
```

## 数据导入导出

### 导出数据

```bash
# 导出所有数据
forgeai data export --output data.json

# 导出特定数据
forgeai data export --type characters --output characters.json
```

### 导入数据

```bash
# 导入数据
forgeai data import --input data.json

# 导入特定数据
forgeai data import --type characters --input characters.json
```
