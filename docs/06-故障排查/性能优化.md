# 性能优化指南

本文档提供ForgeAI性能优化的建议和技巧，帮助您提高创作效率。

---

## API调用优化

### 1. 选择合适的模型

不同模型的速度和成本不同：

```bash
# 快速测试：使用gpt-3.5-turbo
forgeai config set api.model "gpt-3.5-turbo"

# 正式创作：使用gpt-4
forgeai config set api.model "gpt-4"

# 长文本：使用gpt-4-32k
forgeai config set api.model "gpt-4-32k"
```

**模型对比：**
| 模型 | 速度 | 质量 | Token限制 | 成本 |
|------|------|------|-----------|------|
| gpt-3.5-turbo | 快 | 中 | 4K | 低 |
| gpt-4 | 中 | 高 | 8K | 高 |
| gpt-4-32k | 中 | 高 | 32K | 很高 |

### 2. 批量处理减少API调用

```bash
# 批量生成（推荐）
forgeai write chapters 1-10

# 而不是逐章生成（不推荐）
for i in {1..10}; do
  forgeai write chapter $i
done
```

**性能提升：** 批量处理可减少50%以上的API调用次数。

### 3. 控制上下文长度

```bash
# 减少上下文长度（加快生成速度）
forgeai write chapter 1 --context-length 2000

# 默认上下文长度
forgeai write chapter 1 --context-length 3000

# 增加上下文长度（提高连贯性，但速度慢）
forgeai write chapter 1 --context-length 5000
```

**建议：**
- 快速创作：2000-3000 tokens
- 正常创作：3000-4000 tokens
- 高质量创作：4000-5000 tokens

### 4. 调整生成参数

```bash
# 降低temperature（更确定，更快）
forgeai write chapter 1 --temperature 0.5

# 提高temperature（更随机，更慢）
forgeai write chapter 1 --temperature 1.0

# 控制top_p
forgeai write chapter 1 --top-p 0.8
```

## 审查优化

### 1. 选择性使用Agent

不是所有章节都需要所有Agent审查：

```bash
# 日常章节：只使用核心Agent
forgeai review chapter 1 --agents ooc-checker,continuity-checker

# 重要章节：使用所有Agent
forgeai review chapter 50 --all-agents

# 快速检查：只检查严重问题
forgeai review chapter 1 --severity serious
```

**Agent优先级：**
1. **高优先级**：ooc-checker, continuity-checker（影响阅读体验）
2. **中优先级**：reader-pull-checker, pacing-checker（影响追读力）
3. **低优先级**：high-point-checker（仅高潮章节需要）

### 2. 调整Agent敏感度

```yaml
# 降低敏感度（减少误报，加快审查）
agents:
  ooc-checker:
    sensitivity: low
  continuity-checker:
    time_jump_tolerance: 3

# 提高敏感度（更严格，但更慢）
agents:
  ooc-checker:
    sensitivity: high
  continuity-checker:
    time_jump_tolerance: 1
```

### 3. 批量审查和修复

```bash
# 批量审查
forgeai review chapters 1-10

# 批量自动修复
forgeai review fix chapters 1-10 --auto

# 而不是逐章审查修复
for i in {1..10}; do
  forgeai review chapter $i
  forgeai review fix chapter $i --auto
done
```

## 数据管理优化

### 1. 定期清理数据

```bash
# 清理审查缓存
forgeai cleanup --cache

# 清理旧备份
forgeai cleanup --old-backups --keep 5

# 清理临时文件
forgeai cleanup
```

**建议频率：** 每50章清理一次。

### 2. 控制历史数据保留

```yaml
data:
  # 减少历史保留深度
  history_depth: 5  # 默认10

  # 减少备份数量
  backup_retention: 5  # 默认10
```

### 3. 使用数据压缩

```yaml
data:
  # 启用备份压缩
  compress_backup: true
```

## 缓存优化

### 1. 启用上下文缓存

ForgeAI会自动缓存上下文数据，减少重复计算：

```yaml
data:
  # 启用缓存
  enable_cache: true
  
  # 缓存大小限制
  cache_size: 100MB
```

### 2. 预加载常用数据

```bash
# 预加载角色和设定数据
forgeai cache preload --characters
forgeai cache preload --settings
```

## 并行处理优化

### 1. 启用并行生成

```bash
# 并行生成多章
forgeai write chapters 1-10 --parallel --workers 3
```

**注意：** 并行生成会增加API调用频率，注意API速率限制。

### 2. 并行审查

```bash
# 并行审查多章
forgeai review chapters 1-10 --parallel --workers 3
```

## 内存优化

### 1. 分批处理大型项目

```bash
# 分批生成
forgeai write chapters 1-50
forgeai write chapters 51-100
forgeai write chapters 101-150

# 而不是一次性生成
forgeai write chapters 1-150  # 可能内存不足
```

### 2. 减少内存占用

```yaml
data:
  # 减少历史数据保留
  history_depth: 5
  
  # 禁用不必要的数据追踪
  track_all_characters: false
  track_all_locations: false
```

### 3. 使用流式处理

```bash
# 流式生成（减少内存峰值）
forgeai write chapter 1 --stream

# 流式导出
forgeai export --format markdown --stream --output 我的小说.md
```

## 网络优化

### 1. 使用代理或镜像

```bash
# 设置API代理
forgeai config set api.base_url "https://your-proxy.com/v1"

# 或使用镜像
forgeai config set api.base_url "https://api-mirror.com/v1"
```

### 2. 调整超时设置

```yaml
api:
  # 增加超时时间（网络慢时）
  timeout: 60  # 默认30秒
  
  # 重试次数
  max_retries: 3  # 默认2次
  
  # 重试延迟
  retry_delay: 2  # 默认1秒
```

### 3. 启用请求缓存

```yaml
api:
  # 启用请求缓存（重复请求更快）
  enable_cache: true
  
  # 缓存过期时间
  cache_expire: 3600  # 1小时
```

## 存储优化

### 1. 使用SSD存储

将项目放在SSD上可显著提高I/O性能。

### 2. 合理的目录结构

```bash
# 避免过深的目录层级
我的小说/
├── .forgeai/
├── chapters/
└── output/

# 而不是
我的小说/
├── data/
│   ├── forgeai/
│   │   ├── config/
│   │   │   └── ...
```

### 3. 定期整理文件

```bash
# 整理章节文件
forgeai organize chapters

# 整理审查结果
forgeai organize reviews
```

## 优化建议总结

### 快速创作模式

适用于快速测试和初稿创作：

```yaml
api:
  model: "gpt-3.5-turbo"
  
writing:
  context_length: 2000
  
review:
  agents:
    - ooc-checker
    - continuity-checker
  severity_threshold: "serious"
  
data:
  history_depth: 3
```

### 标准创作模式

适用于正常创作：

```yaml
api:
  model: "gpt-4"
  
writing:
  context_length: 3000
  
review:
  agents:
    - ooc-checker
    - continuity-checker
    - reader-pull-checker
    - pacing-checker
  severity_threshold: "moderate"
  
data:
  history_depth: 5
```

### 高质量创作模式

适用于最终定稿：

```yaml
api:
  model: "gpt-4"
  
writing:
  context_length: 5000
  
review:
  agents:
    - consistency-checker
    - ooc-checker
    - continuity-checker
    - reader-pull-checker
    - high-point-checker
    - pacing-checker
  severity_threshold: "minor"
  
data:
  history_depth: 10
```

## 性能基准

在不同配置下的预期性能：

| 配置 | 生成速度 | 审查速度 | 内存占用 | API成本 |
|------|----------|----------|----------|---------|
| 快速模式 | 30章/小时 | 60章/小时 | <500MB | 低 |
| 标准模式 | 15章/小时 | 30章/小时 | ~1GB | 中 |
| 高质量模式 | 8章/小时 | 15章/小时 | ~2GB | 高 |

**注：** 实际性能取决于硬件、网络和API服务。
