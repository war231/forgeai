# 常见错误及解决方案

本文档列出使用ForgeAI时可能遇到的常见错误及其解决方案。

---

## 安装和配置错误

### 错误1: API密钥无效

**错误信息：**
```
APIError: Invalid API key provided
```

**原因：**
- API密钥错误或过期
- API密钥格式不正确

**解决方案：**
```bash
# 检查API密钥
forgeai config show

# 重新设置API密钥
forgeai config set api.key "your-correct-api-key"

# 测试API连接
forgeai config test-api
```

### 错误2: API连接失败

**错误信息：**
```
ConnectionError: Failed to connect to API endpoint
```

**原因：**
- 网络连接问题
- API服务不可用
- 代理设置错误

**解决方案：**
```bash
# 检查网络连接
ping api.openai.com

# 检查API基础URL
forgeai config show

# 如果使用代理，设置正确的base_url
forgeai config set api.base_url "https://your-proxy-url/v1"

# 测试API连接
forgeai config test-api
```

### 错误3: 模型不可用

**错误信息：**
```
ModelError: Model 'gpt-4' is not available
```

**原因：**
- 模型名称错误
- API账户无权限访问该模型

**解决方案：**
```bash
# 检查可用模型
forgeai config test-api --list-models

# 使用可用模型
forgeai config set api.model "gpt-3.5-turbo"
```

### 错误4: 配置文件损坏

**错误信息：**
```
ConfigError: Failed to parse config.yaml
```

**原因：**
- YAML格式错误
- 配置文件缺失必需字段

**解决方案：**
```bash
# 验证配置文件
forgeai config check

# 重置配置文件
forgeai config reset

# 或手动编辑修复
forgeai config edit
```

## 项目创建错误

### 错误5: 项目已存在

**错误信息：**
```
ProjectError: Project '我的小说' already exists
```

**原因：**
- 同名项目已存在

**解决方案：**
```bash
# 使用不同名称
forgeai init 新小说

# 或删除现有项目
forgeai delete 我的小说
forgeai init 我的小说
```

### 错误6: 项目路径无效

**错误信息：**
```
PathError: Invalid project path
```

**原因：**
- 路径不存在或无权限

**解决方案：**
```bash
# 创建目录
mkdir -p path/to/project

# 检查权限
ls -la path/to/project

# 使用绝对路径
forgeai init 我的小说 --path /absolute/path/to/project
```

## 创作错误

### 错误7: Token超出限制

**错误信息：**
```
TokenLimitError: Token limit exceeded (max: 4096, used: 4500)
```

**原因：**
- 上下文过长，超出模型token限制

**解决方案：**
```bash
# 减少上下文长度
forgeai write chapter 1 --context-length 3000

# 使用更大token限制的模型
forgeai config set api.model "gpt-4-32k"

# 分段生成
forgeai write chapter 1 --split
```

### 错误8: 生成内容为空

**错误信息：**
```
GenerationError: Generated content is empty
```

**原因：**
- 提示词不当
- 模型未正确响应

**解决方案：**
```bash
# 检查提示词
forgeai show prompt --chapter 1

# 调整参数
forgeai write chapter 1 --temperature 0.8

# 重新生成
forgeai write chapter 1 --regenerate
```

### 错误9: 章节内容不符合预期

**错误信息：**
无错误信息，但生成内容不符合预期

**原因：**
- 风格设置不当
- 上下文不足
- 参数设置不合理

**解决方案：**
```bash
# 明确指定风格
forgeai write chapter 1 --style "更紧张悬疑"

# 增加上下文
forgeai write chapter 1 --context-chapters 5

# 调整参数
forgeai write chapter 1 \
  --temperature 0.7 \
  --top-p 0.9 \
  --words 3000

# 使用样章
forgeai write chapter 1 --template sample.md
```

## 审查错误

### 错误10: Agent执行失败

**错误信息：**
```
AgentError: Agent 'ooc-checker' failed to execute
```

**原因：**
- Agent配置错误
- 数据格式不正确

**解决方案：**
```bash
# 检查Agent配置
forgeai config show --section agents

# 验证数据
forgeai validate all

# 使用单个Agent测试
forgeai review chapter 1 --agent ooc-checker --verbose
```

### 错误11: 审查结果异常

**错误信息：**
无错误信息，但审查结果异常（如问题过多或过少）

**原因：**
- 审查标准设置不当
- Agent敏感度不合理

**解决方案：**
```bash
# 调整敏感度
forgeai config set agents.ooc-checker.sensitivity low

# 调整严重度阈值
forgeai review chapter 1 --severity serious

# 查看详细审查过程
forgeai review chapter 1 --verbose
```

### 错误12: 自动修复失败

**错误信息：**
```
FixError: Failed to auto-fix issue
```

**原因：**
- 问题过于复杂，无法自动修复
- 修复逻辑不适用

**解决方案：**
```bash
# 手动修复
forgeai review fix chapter 1 --interactive

# 或编辑章节
forgeai edit chapter 1

# 重新审查
forgeai review chapter 1
```

## 数据错误

### 错误13: 角色数据损坏

**错误信息：**
```
DataError: Failed to load character '李明'
```

**原因：**
- 角色文件格式错误
- 文件损坏或缺失

**解决方案：**
```bash
# 验证角色数据
forgeai validate characters

# 查看角色文件
forgeai character show 李明 --verbose

# 重新创建角色
forgeai character delete 李明
forgeai character add 李明 --age 25 --gender 男

# 或从备份恢复
forgeai backup restore --name "备份名称"
```

### 错误14: 时间线数据冲突

**错误信息：**
```
TimelineError: Timeline conflict detected
```

**原因：**
- 时间线数据不一致
- 时间跳跃逻辑错误

**解决方案：**
```bash
# 检查时间线
forgeai timeline check

# 查看时间线详情
forgeai timeline view --verbose

# 更新时间线
forgeai timeline update

# 或重置时间线
forgeai timeline reset
```

### 错误15: 上下文数据丢失

**错误信息：**
```
ContextError: Context data is missing or corrupted
```

**原因：**
- 上下文文件损坏
- 数据未正确保存

**解决方案：**
```bash
# 检查上下文
forgeai context view

# 更新上下文
forgeai context update

# 或重置上下文
forgeai context reset

# 从备份恢复
forgeai backup restore --name "备份名称"
```

## 导出错误

### 错误16: 导出格式不支持

**错误信息：**
```
ExportError: Unsupported format 'xyz'
```

**原因：**
- 导出格式名称错误
- 导出工具未安装

**解决方案：**
```bash
# 查看支持的格式
forgeai export --help

# 使用支持的格式
forgeai export --format markdown --output 我的小说.md
forgeai export --format docx --output 我的小说.docx
forgeai export --format pdf --output 我的小说.pdf
forgeai export --format epub --output 我的小说.epub
```

### 错误17: 导出文件过大

**错误信息：**
```
ExportError: Output file too large
```

**原因：**
- 内容过多，单个文件过大

**解决方案：**
```bash
# 分卷导出
forgeai export --format markdown --split-by 50 --output 卷

# 按三幕结构分卷
forgeai export --format markdown --split-by-act --output 卷

# 导出为多个章节文件
forgeai export --format markdown --separate-chapters --output chapters/
```

## 性能错误

### 错误18: 生成速度过慢

**错误信息：**
无错误信息，但生成速度过慢

**原因：**
- 模型响应慢
- 网络延迟
- 上下文过长

**解决方案：**
```bash
# 使用更快的模型
forgeai config set api.model "gpt-3.5-turbo"

# 减少上下文长度
forgeai write chapter 1 --context-length 2000

# 批量生成（减少API调用次数）
forgeai write chapters 1-10 --batch
```

### 错误19: 内存占用过高

**错误信息：**
```
MemoryError: Out of memory
```

**原因：**
- 项目数据过大
- 内存不足

**解决方案：**
```bash
# 清理缓存
forgeai cleanup --cache

# 分批处理
forgeai write chapters 1-5
forgeai write chapters 6-10

# 减少保留的历史数据
forgeai config set data.history_depth 5
```

## 备份错误

### 错误20: 备份恢复失败

**错误信息：**
```
BackupError: Failed to restore backup
```

**原因：**
- 备份文件损坏
- 版本不兼容

**解决方案：**
```bash
# 查看备份列表
forgeai backup list

# 查看备份详情
forgeai backup show --name "备份名称"

# 尝试其他备份
forgeai backup restore --name "其他备份"

# 或手动恢复
cp -r .forgeai/backups/备份名称/* .forgeai/
```

## 通用错误处理流程

当遇到未列出的错误时，按以下流程处理：

### 1. 查看详细错误信息

```bash
# 启用详细日志
export FORGEAI_LOG_LEVEL=DEBUG

# 重新执行命令
forgeai <command> --verbose
```

### 2. 检查配置和数据

```bash
# 验证配置
forgeai config check

# 验证数据
forgeai validate all

# 测试API连接
forgeai config test-api
```

### 3. 尝试重置

```bash
# 重置配置
forgeai config reset

# 重置上下文
forgeai context reset

# 清理缓存
forgeai cleanup --all
```

### 4. 从备份恢复

```bash
# 查看备份
forgeai backup list

# 恢复最近的备份
forgeai backup restore --name "最近的备份"
```

### 5. 重新安装

```bash
# 卸载
pip uninstall forgeai

# 清理缓存
rm -rf ~/.forgeai

# 重新安装
pip install forgeai
```
