# 故障排查最佳实践

本文档提供ForgeAI故障排查的最佳实践和建议，帮助您快速定位和解决问题。

---

## 预防性最佳实践

### 1. 定期备份

**重要性：** 备份是防止数据丢失的最后一道防线。

**实践建议：**
```bash
# 每50章备份一次
forgeai backup create --name "进度备份-第50章" --tag "v0.5"

# 重要修改前备份
forgeai backup create --name "修改前备份"

# 完成后备份
forgeai backup create --name "第一版完成" --tag "v1.0"
```

**备份策略：**
- 自动备份：每天一次
- 手动备份：每50章一次
- 重要节点：修改前、完成后

### 2. 定期验证

**重要性：** 定期验证可及早发现问题。

**实践建议：**
```bash
# 每天验证一次
forgeai validate all

# 验证特定数据
forgeai validate characters
forgeai validate settings
forgeai validate plots

# 验证配置
forgeai config check
```

### 3. 监控日志

**重要性：** 日志可帮助定位问题。

**实践建议：**
```bash
# 查看最新日志
tail -f .forgeai/logs/forgeai.log

# 搜索错误日志
grep "ERROR" .forgeai/logs/forgeai.log

# 查看特定时间日志
grep "2024-01-15" .forgeai/logs/forgeai.log
```

## 问题定位最佳实践

### 1. 启用详细日志

当遇到问题时，首先启用详细日志：

```bash
# 设置日志级别
export FORGEAI_LOG_LEVEL=DEBUG

# 重新执行命令
forgeai <command> --verbose

# 查看详细日志
tail -f .forgeai/logs/forgeai.log
```

### 2. 分步排查

将复杂操作分解为简单步骤：

```bash
# 不要一次性执行
forgeai write chapters 1-100 --auto-review --auto-fix

# 而是分步执行
forgeai write chapter 1           # 第1步：生成
forgeai review chapter 1          # 第2步：审查
forgeai review fix chapter 1      # 第3步：修复
forgeai validate all              # 第4步：验证
```

### 3. 隔离问题

尝试在最小环境中重现问题：

```bash
# 创建测试项目
forgeai init 测试项目

# 只执行问题命令
forgeai <problematic-command>

# 如果问题重现，说明是ForgeAI问题
# 如果问题不重现，说明是项目数据问题
```

### 4. 检查环境

确认环境配置正确：

```bash
# 检查Python版本
python --version  # 建议3.8+

# 检查ForgeAI版本
forgeai --version

# 检查依赖
pip list | grep forgeai

# 检查系统资源
free -h  # 内存
df -h    # 磁盘
```

## 问题解决最佳实践

### 1. 从简单到复杂

按以下顺序尝试解决：

**Level 1: 重试**
```bash
# 简单重试
forgeai <command>
```

**Level 2: 清理缓存**
```bash
# 清理缓存
forgeai cleanup --cache

# 重试
forgeai <command>
```

**Level 3: 重置状态**
```bash
# 重置上下文
forgeai context reset

# 重试
forgeai <command>
```

**Level 4: 从备份恢复**
```bash
# 查看备份
forgeai backup list

# 恢复备份
forgeai backup restore --name "最近的备份"
```

**Level 5: 重新安装**
```bash
# 卸载
pip uninstall forgeai

# 清理
rm -rf ~/.forgeai

# 重新安装
pip install forgeai
```

### 2. 保留现场

在尝试修复前，保留问题现场：

```bash
# 导出错误日志
cp .forgeai/logs/forgeai.log error-$(date +%Y%m%d).log

# 导出配置
forgeai config show > config-backup.yaml

# 创建问题备份
forgeai backup create --name "问题现场"
```

### 3. 记录问题

记录问题详情，便于分析和求助：

```markdown
## 问题描述

**时间：** 2024-01-15 10:30
**命令：** forgeai write chapter 1
**错误：** [完整错误信息]

## 环境

- ForgeAI版本：1.0.0
- Python版本：3.10.0
- 操作系统：Windows 11
- API模型：gpt-4

## 复现步骤

1. 创建项目
2. 添加角色
3. 执行命令

## 尝试的解决方案

1. 重试 - 无效
2. 清理缓存 - 无效
3. 从备份恢复 - 有效
```

## 常见问题排查流程

### API问题排查

```bash
# 1. 测试API连接
forgeai config test-api

# 2. 检查API密钥
forgeai config show | grep api.key

# 3. 检查API模型
forgeai config show | grep api.model

# 4. 查看API调用日志
grep "API" .forgeai/logs/forgeai.log

# 5. 尝试不同模型
forgeai config set api.model "gpt-3.5-turbo"
forgeai config test-api
```

### 数据问题排查

```bash
# 1. 验证数据
forgeai validate all

# 2. 检查数据文件
ls -la .forgeai/

# 3. 查看数据内容
forgeai context view
forgeai timeline view

# 4. 检查数据格式
file .forgeai/*.json

# 5. 尝试修复数据
forgeai data repair
```

### 审查问题排查

```bash
# 1. 单独测试Agent
forgeai review chapter 1 --agent ooc-checker --verbose

# 2. 检查Agent配置
forgeai config show --section agents

# 3. 降低敏感度
forgeai config set agents.ooc-checker.sensitivity low

# 4. 查看审查日志
grep "review" .forgeai/logs/forgeai.log

# 5. 手动审查
forgeai edit chapter 1
```

### 性能问题排查

```bash
# 1. 查看性能统计
forgeai stats performance

# 2. 检查内存使用
forgeai stats memory

# 3. 检查API调用次数
forgeai stats api

# 4. 查看慢操作日志
grep "slow" .forgeai/logs/forgeai.log

# 5. 优化配置
forgeai config set api.model "gpt-3.5-turbo"
```

## 最佳实践总结

### DO（推荐做法）

✅ 定期备份（每50章）
✅ 定期验证（每天）
✅ 启用详细日志（遇到问题时）
✅ 分步排查问题
✅ 保留问题现场
✅ 记录问题详情
✅ 搜索已有问题
✅ 提供完整信息求助
✅ 定期更新和清理

### DON'T（不推荐做法）

❌ 忽略警告信息
❌ 一次性执行复杂操作
❌ 在无备份情况下修改
❌ 删除日志文件
❌ 修改内部数据文件
❌ 使用不支持的配置
❌ 忽略性能问题
❌ 在问题未解决时继续创作
