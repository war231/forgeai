# 章节创作

章节创作是ForgeAI工作流的核心环节。本文档详细介绍如何使用ForgeAI高效创作章节内容，包括生成、审查、优化等完整流程。

## 创作流程概览

```
┌─────────────────────────────────────┐
│         章节创作流程                 │
├─────────────────────────────────────┤
│                                     │
│  1. 准备上下文                      │
│     ├─ 读取前文内容                 │
│     ├─ 加载角色状态                 │
│     └─ 确认情节进度                 │

│  2. 生成章节                        │
```

## 1. 准备上下文

### 1.1 查看当前状态

```bash
# 查看项目状态
forgeai status

# 查看当前进度
forgeai progress

# 查看上下文摘要
forgeai context summary
```

## 2. 生成章节

### 2.1 基本生成

```bash
# 生成下一章
forgeai write next

# 生成特定章节
forgeai write chapter 11

# 批量生成
forgeai write chapters 11-20

# 继续创作（从最后一章开始）
forgeai write continue
```

### 2.2 高级选项

```bash
# 指定字数
forgeai write chapter 11 --words 3000

# 指定风格
forgeai write chapter 11 --style "更紧张悬疑"

# 指定视角
forgeai write chapter 11 --perspective "李明"

# 指定场景
forgeai write chapter 11 --scene "公司办公室"

# 指定情节点
forgeai write chapter 11 --plot-point "李明开始调查"
```

```
  场景2: 李明家 (晚上)
```

```bash
# 查看章节摘要
forgeai show chapter 11 --summary

# 编辑章节
forgeai edit chapter 11
```

## 3. 审查章节

### 3.1 运行审查

```bash
# 审查单章
forgeai review chapter 11

# 使用特定Agent
forgeai review chapter 11 --agent ooc-checker

# 使用多个Agent
forgeai review chapter 11 --agents ooc-checker,continuity-checker

# 使用所有Agent
forgeai review chapter 11 --all-agents

# 批量审查
forgeai review chapters 11-20
```

## 4. 处理问题

### 4.1 自动修复

```bash
# 自动修复所有问题
forgeai review fix chapter 11 --auto

# 自动修复特定问题
forgeai review fix chapter 11 --issue 1

# 自动修复中等及以上问题
forgeai review fix chapter 11 --severity moderate
```

**自动修复日志：**
```

[1/2] 修复OOC问题
  问题: 李明主动与陌生同事搭话

    "李明正专注工作，旁边的新同事主动搭话..."
  ✓ 修复完成

[2/2] 修复连贯性问题

    "到了晚上，李明回到家中..."
  ✓ 修复完成
```

### 4.2 手动修改

```bash
# 交互式修复
forgeai review fix chapter 11 --interactive

# 打开编辑器
forgeai edit chapter 11

# 手动修改后重新审查
forgeai review chapter 11
```

**交互式修复示例：**

### 4.3 忽略问题

```bash
# 忽略特定问题
forgeai review ignore chapter 11 --issue 2

# 忽略轻微问题
forgeai review ignore chapter 11 --severity minor

# 查看忽略的问题
forgeai review show chapter 11 --ignored
```

## 5. 迭代优化

### 5.1 重新生成

```bash
# 完全重新生成
forgeai write chapter 11 --regenerate

# 重新生成特定场景
forgeai write chapter 11 --regenerate-scene 2

# 重新生成特定段落
forgeai write chapter 11 --regenerate-paragraph 5
```

### 5.2 局部优化

```bash
# 优化对话
forgeai optimize chapter 11 --focus dialogue

# 优化描写
forgeai optimize chapter 11 --focus description

# 优化节奏
forgeai optimize chapter 11 --focus pacing

# 优化追读力
forgeai optimize chapter 11 --focus reader-pull
```

### 5.3 调整参数

```bash
# 调整写作风格
forgeai write chapter 11 --style "更紧张悬疑"

# 调整章节长度
forgeai write chapter 11 --words 3000

# 调整描写比例
forgeai write chapter 11 --description-ratio "action:0.4,dialogue:0.3,psychology:0.2"

# 调整视角
forgeai write chapter 11 --perspective "李明"
```

### 5.4 更新上下文

```bash
# 手动更新上下文
forgeai context update

# 更新角色状态
forgeai context update --character 李明 --location 上海

# 更新情节进度
forgeai context update --plot main --progress 0.15
```

## 创作技巧

### 1. 保持上下文连贯

```bash
# 每次创作前检查上下文
forgeai context summary

# 确认角色状态
forgeai show character 李明

# 确认情节进度
forgeai show plot main
```

### 2. 定期审查

```bash
# 每完成一章立即审查
forgeai write next && forgeai review last

# 每5章进行一次全面审查
forgeai review chapters 1-5 --all-agents
```

### 3. 关注追读力

```bash
# 检查追读力评分
forgeai review chapter 11 --agent reader-pull-checker

# 优化追读力
forgeai optimize chapter 11 --focus reader-pull
```

### 4. 保持节奏

```bash
# 检查节奏
forgeai review chapter 11 --agent pacing-checker

# 查看节奏曲线
forgeai plot pacing --chapters 1-20
```

### 5. 及时备份

```bash
# 每完成5章备份一次
forgeai backup create --name "进度备份-第20章"
```

## 常见问题

### Q: 生成的章节不符合预期怎么办？

A: 可以：
1. 使用 `--regenerate` 重新生成
2. 使用 `--style` 调整风格
3. 使用 `--plot-point` 指定情节点
4. 手动编辑后继续

### Q: 审查发现问题太多怎么办？

A: 建议：
1. 优先处理严重和中等问题
2. 使用自动修复处理简单问题
3. 忽略不影响阅读的轻微问题
4. 重新生成问题过多的章节

### Q: 如何提高生成质量？

A: 建议：
1. 完善写前准备（设定、大纲）
2. 使用样章模板
3. 明确指定情节点和场景
4. 及时审查和优化

### Q: 批量创作如何保证质量？

A: 建议：
1. 使用 `--auto-review` 自动审查
2. 使用 `--auto-fix` 自动修复
3. 定期人工复核
4. 关注质量评分趋势

## 最佳实践

1. **创作前检查**：确认上下文和设定
2. **立即审查**：每章创作后立即审查
3. **优先修复**：优先处理严重问题
4. **定期优化**：每5-10章进行一次优化
5. **及时备份**：重要节点创建备份
6. **关注质量**：追读力和一致性比速度更重要
