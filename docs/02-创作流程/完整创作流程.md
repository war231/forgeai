## 第一阶段：项目初始化

### 1.1 创建项目

```bash
# 创建新项目
forgeai init 我的小说

# 进入项目目录
cd 我的小说
```

**生成的项目结构：**
```
我的小说/
├── .forgeai/              # ForgeAI配置目录
│   ├── config.yaml        # 项目配置
│   ├── characters/        # 角色设定
│   ├── settings/          # 世界设定
│   ├── timeline.json      # 时间线数据
│   └── context.json       # 上下文数据
├── chapters/              # 章节内容
│   └── chapter-001.md
├── output/                # 输出目录
└── README.md              # 项目说明
```

### 1.2 配置API密钥

```bash
# 方式1: 命令行配置
forgeai config set api.key "your-api-key"
forgeai config set api.model "gpt-4"

# 方式2: 编辑配置文件
# 编辑 .forgeai/config.yaml
```

**配置示例：**
```yaml
api:
  provider: openai
  key: "your-api-key"
  model: "gpt-4"
  base_url: "https://api.openai.com/v1"

project:
  name: "我的小说"
  genre: "都市"
  target_words: 500000
  target_chapters: 200

writing:
  words_per_chapter: 2500
  style: "轻松幽默"
  perspective: "第三人称"

review:
  auto_review: true
  agents:
    - consistency-checker
    - ooc-checker
    - continuity-checker
    - reader-pull-checker
```

### 1.3 验证配置

```bash
# 检查配置
forgeai config check

# 测试API连接
forgeai config test-api
```

## 第二阶段：写前准备

### 2.1 定义世界观

创建世界设定文件 `.forgeai/settings/世界观.yaml`：

```yaml
name: 现代都市
time_period: 2024年
technology_level: 现代

world_rules:
  - "无超自然元素"
  - "遵循现实物理法则"
  - "科技水平与现实一致"

locations:
  北京:
    type: 城市
    climate: 温带季风
    districts:
      - 朝阳区
      - 海淀区
      - 西城区
    landmarks:
      - 天安门
      - 故宫
      - 颐和园

  上海:
    type: 城市
    climate: 亚热带季风
    districts:
      - 浦东新区
      - 黄浦区
    landmarks:
      - 外滩
      - 东方明珠
```

### 2.2 创建角色设定

创建角色文件 `.forgeai/characters/李明.yaml`：

```yaml
name: 李明
age: 25
gender: 男
occupation: 程序员

appearance:
  height: 175cm
  build: 偏瘦
  features:
    - 戴眼镜
    - 黑色短发
    - 目光锐利

personality:
  traits:
    - 谨慎
    - 聪明
    - 内向
    - 责任心强
  strengths:
    - 逻辑思维强
    - 观察力敏锐
    - 学习能力强
  weaknesses:
    - 社交能力弱
    - 过度谨慎
    - 不善表达情感

background: |
  李明从小对计算机感兴趣，大学计算机系毕业后，
  进入一家互联网公司担任后端工程师。
  性格内向，朋友不多，但工作能力出色。

abilities:
  编程: 9
  逻辑思维: 8
  观察: 8
  社交: 3
  运动: 4

relationships:
  王芳: 朋友
  张刚: 同事
  李总: 上司
```

### 2.3 规划情节大纲

创建情节大纲文件 `.forgeai/plots/main-plot.yaml`：

```yaml
name: 主线情节
genre: 都市职场

structure:
  first_act:
    chapters: 1-50
    theme: "新人入职，适应环境"
    key_events:
      - chapter: 1
        event: "李明入职新公司"
      - chapter: 10

      - chapter: 30

    chapters: 51-150

    key_events:
      - chapter: 60
        event: "收集证据"
      - chapter: 100
        event: "遭遇挫折"
      - chapter: 120
        event: "获得关键线索"
      - chapter: 150
        event: "真相大白"

  third_act:
    chapters: 151-200

    key_events:
      - chapter: 160
        event: "对抗反派"
      - chapter: 180
        event: "解决问题"
      - chapter: 200
        event: "新的开始"

characters:
  protagonists:
    - 李明
    - 王芳
  antagonists:
    - 神秘人
  supporting:
    - 张刚
    - 李总
```

```bash
# 导入样章（可选）

# 或创建样章模板
```

## 第三阶段：章节创作

### 3.1 生成章节内容

```bash
# 生成单章
forgeai write chapter 1

# 批量生成
forgeai write chapters 1-10

# 继续创作（从最后一章开始）
forgeai write continue
```

**创作过程：**
```
1. 读取上下文（前文内容、角色状态、情节进度）
2. 生成章节大纲
3. 生成章节内容
4. 自动保存到 chapters/chapter-XXX.md
5. 更新上下文数据
```

### 3.2 运行审查Agent

```bash
# 审查单章
forgeai review chapter 1

# 审查多章
forgeai review chapters 1-10

# 使用特定Agent
forgeai review chapter 1 --agent ooc-checker

# 使用所有Agent
forgeai review chapter 1 --all-agents
```

**审查流程：**
```
┌─────────────────────────────────────┐
│         审查Agent工作流              │
├─────────────────────────────────────┤
│                                     │
│  1. 设定一致性检查                   │
│     └─ 检查角色设定是否矛盾          │
│                                     │
│  2. OOC检测                         │
│     └─ 检查角色行为是否符合设定      │
│                                     │
│  3. 连贯性检查                       │
│     └─ 检查情节逻辑是否连贯          │
│                                     │
│  4. 追读力检查                       │
│     └─ 评估追读力和弃书风险          │
│                                     │
│  5. 高潮点检查                       │
│     └─ 检查高潮场景设置              │
│                                     │
│  6. 节奏检查                         │
│     └─ 评估叙事节奏                  │
│                                     │
└─────────────────────────────────────┘
```

### 3.3 处理审查问题

```bash
# 查看审查结果
forgeai review show chapter 1

# 自动修复问题
forgeai review fix chapter 1 --auto

# 手动修复（交互式）
forgeai review fix chapter 1 --interactive

# 忽略特定问题
forgeai review ignore chapter 1 --issue 3
```

**审查结果示例：**
```
章节: 第1章
审查时间: 2024-01-15 10:30:00

问题汇总:
  严重: 0
  中等: 2
  轻微: 3

问题详情:
[中等] OOC检测 - 第3段
  角色: 李明
  问题: 单手举起50斤重物，超出能力设定
  建议: 改为两人一起搬，或使用工具

[中等] 连贯性检查 - 第5段→第6段
  问题: 时间从周一直接跳到周五，无过渡
  建议: 添加时间标记，如"四天后"

[轻微] 追读力检查 - 章节结尾
  问题: 结尾平淡，无悬念钩子
  建议: 添加悬念元素
```

### 3.4 迭代优化

```bash
# 重新生成章节
forgeai write chapter 1 --regenerate

# 基于审查结果优化
forgeai optimize chapter 1 --based-on-review

# 调整写作风格
forgeai write chapter 1 --style "更轻松幽默"

# 调整章节长度
forgeai write chapter 1 --words 3000
```

## 第四阶段：写后处理

### 4.1 全书一致性检查

```bash
# 全书一致性检查
forgeai review all --consistency

# 检查特定维度
forgeai review all --focus timeline
forgeai review all --focus character
forgeai review all --focus plot

# 生成一致性报告
forgeai report consistency --output consistency-report.md
```

**一致性检查内容：**
- 角色设定一致性
- 时间线连贯性
- 情节逻辑一致性
- 前后呼应一致性

```bash
forgeai report stats
```

## 推荐工作流程

### 日常创作流程

```bash
# 1. 开始创作
forgeai write continue

# 2. 审查新章节
forgeai review last

# 3. 处理问题
forgeai review fix last --interactive

# 4. 查看进度
forgeai status

# 5. 定期备份
forgeai backup create --name "进度备份"
```

### 批量创作流程

```bash
# 1. 批量生成
forgeai write chapters 51-100

# 2. 批量审查
forgeai review chapters 51-100

# 3. 批量修复
forgeai review fix chapters 51-100 --auto

# 4. 人工复核
forgeai review chapters 51-100 --severity serious

# 5. 生成报告
forgeai report stats --chapters 51-100
```

### 质量检查流程

```bash
# 1. 全书一致性检查
forgeai review all --consistency

# 2. 追读力评估
forgeai review all --agent reader-pull-checker

# 3. 生成质量报告
forgeai report quality --output quality-report.md

# 4. 优化建议
forgeai suggest improvements

# 5. 执行优化
forgeai optimize all --based-on-report
```

## 常见问题

### Q: 如何处理创作瓶颈？
A: ForgeAI提供多种辅助：
- 查看情节大纲，了解下一步方向
- 使用RAG检索相关素材和灵感
- 生成多个版本，选择最佳方案
- 调整写作参数，尝试不同风格

### Q: 如何保证创作质量？
A: 建议采用以下策略：
- 每章创作后立即审查
- 定期进行全书一致性检查
- 关注追读力评分和弃书风险
- 根据审查建议持续优化

### Q: 如何提高创作效率？
A: 推荐做法：
- 完善写前准备（设定、大纲）
- 使用批量创作功能
- 启用自动审查和自动修复
- 定期备份，避免重复工作

### Q: 如何处理大量审查问题？
A: 建议策略：
- 优先处理严重和中等问题
- 使用自动修复处理简单问题
- 人工复核重要问题
- 忽略不影响阅读的轻微问题

## 最佳实践

1. **充分准备**：写前准备越充分，创作越顺利
2. **持续审查**：每章创作后立即审查，避免问题累积
3. **定期备份**：重要节点创建备份，防止数据丢失
4. **关注质量**：追读力和一致性比速度更重要
5. **灵活调整**：根据审查结果调整创作策略
6. **保持连贯**：定期检查全书一致性
