# 最佳实践

本文档总结了使用ForgeAI进行长篇小说创作的最佳实践，帮助您提高创作效率和质量。

## 创作流程最佳实践

### 1. 充分的写前准备

**重要性：** 写前准备越充分，创作越顺利，后期修改越少。

**实践建议：**
- 详细定义世界观和规则
- 完整创建主要角色设定
- 规划清晰的三幕结构
- 标注关键情节点和转折点
- 准备样章和风格指南

**检查清单：**
- [ ] 世界观已定义
- [ ] 主要角色已创建（至少1个）
- [ ] 主线大纲已规划
- [ ] 关键节点已标注
- [ ] 写作风格已设定

### 2. 持续的章节审查

**重要性：** 每章创作后立即审查，避免问题累积。

**实践建议：**
- 每章创作后立即运行审查
- 优先处理严重和中等问题
- 使用自动修复处理简单问题
- 人工复核重要问题
- 忽略不影响阅读的轻微问题

**推荐工作流：**
```bash
# 创作并审查
forgeai write next
forgeai review last

# 处理问题
forgeai review fix last --auto
forgeai review fix last --interactive  # 处理复杂问题
```

### 3. 定期的质量检查

**重要性：** 定期检查整体质量，及时发现和修复问题。

**实践建议：**
- 每5-10章进行一次质量检查
- 关注追读力评分和弃书风险
- 检查节奏和高潮分布
- 生成质量报告并分析

**推荐工作流：**
```bash
# 每10章检查一次
forgeai review chapters 1-10 --all-agents
forgeai report quality --chapters 1-10
```

### 4. 及时的备份策略

**重要性：** 定期备份防止数据丢失，保留创作历史。

**实践建议：**
- 重要节点创建备份（如每50章）
- 使用有意义的备份名称和标签
- 定期清理旧备份（保留最近5-10个）
- 导出项目完整备份到安全位置

**推荐工作流：**
```bash
# 每50章备份一次
forgeai backup create --name "进度备份-第50章" --tag "v0.5"

# 完成后创建最终备份
forgeai backup create --name "第一版完成" --tag "v1.0"
```

## 质量提升最佳实践

### 1. 保持追读力

**关键要素：**
- 每章设置悬念钩子
- 保持悬念密度（每2-3章新悬念）
- 控制信息释放节奏
- 维持冲突强度

**检查方法：**
```bash
# 检查追读力
forgeai review chapter 11 --agent reader-pull-checker

# 查看追读力曲线
forgeai plot reader-pull --chapters 1-20
```

### 2. 控制叙事节奏

**关键要素：**
- 根据场景类型选择节奏
- 快慢节奏交替，避免单调
- 保持均衡的描写比例
- 长短段落交替

**检查方法：**
```bash
# 检查节奏
forgeai review chapter 11 --agent pacing-checker

# 查看节奏曲线
forgeai plot pacing --chapters 1-20
```

### 3. 合理设置高潮

**关键要素：**
- 按三幕结构分布高潮点
- 高潮强度递进上升
- 高潮前充分铺垫
- 高潮后处理余波

**检查方法：**
```bash
# 检查高潮分布
forgeai review all --agent high-point-checker

# 查看高潮分布图
forgeai plot high-points
```

### 4. 维护一致性

**关键要素：**
- 角色行为符合设定
- 时间线连贯清晰
- 情节逻辑合理
- 前后呼应一致

**检查方法：**
```bash
# 检查一致性
forgeai review all --consistency

# 生成一致性报告
forgeai report consistency
```

## 效率提升最佳实践

### 1. 使用批量功能

**适用场景：**
- 批量生成多章
- 批量审查和修复
- 批量优化调整

**示例：**
```bash
# 批量生成并自动审查修复
forgeai write chapters 11-20 --auto-review --auto-fix

# 批量优化风格
forgeai optimize chapters 11-20 --style "更轻松幽默"
```

### 2. 使用自动修复

**适用场景：**
- 简单的OOC问题
- 时间跳跃未说明
- 描写比例失衡

**示例：**
```bash
# 自动修复所有问题
forgeai review fix chapter 11 --auto

# 自动修复中等及以上问题
forgeai review fix chapter 11 --severity moderate
```

### 3. 使用模板和配置

**适用场景：**
- 创建相似项目
- 保持风格一致
- 快速开始创作

**示例：**
```bash
# 从模板创建项目
forgeai init 新小说 --template 都市职场

# 使用配置预设
forgeai config apply --preset 轻松幽默
```

### 4. 使用RAG检索

**适用场景：**
- 查找相似情节
- 检索角色信息
- 获取创作灵感

**示例：**
```bash
# 检索相似场景
forgeai search "职场冲突场景"

# 检索角色信息
forgeai search "李明的性格特点"
```

## 问题处理最佳实践

### 1. OOC问题处理

**处理策略：**
1. 检查角色设定是否清晰
2. 分析OOC原因（设定不清/创作偏离）
3. 使用自动修复或手动调整
4. 更新角色设定（如需要）

**示例：**
```bash
# 检查OOC
forgeai review chapter 11 --agent ooc-checker

# 自动修复
forgeai review fix chapter 11 --agent ooc-checker --auto
```

### 2. 连贯性问题处理

**处理策略：**
1. 检查时间线数据
2. 添加时间标记或过渡段落
3. 调整情节顺序（如需要）
4. 更新时间线数据

**示例：**
```bash
# 检查连贯性
forgeai review chapter 11 --agent continuity-checker

# 查看时间线
forgeai timeline view

# 更新时间线
forgeai timeline update
```

### 3. 追读力问题处理

**处理策略：**
1. 检查章节钩子强度
2. 添加悬念元素
3. 调整信息释放节奏
4. 优化冲突强度

**示例：**
```bash
# 检查追读力
forgeai review chapter 11 --agent reader-pull-checker

# 优化追读力
forgeai optimize chapter 11 --focus reader-pull
```

### 4. 节奏问题处理

**处理策略：**
1. 检查场景类型和节奏匹配
2. 调整描写比例
3. 优化段落长度
4. 插入或删除场景

**示例：**
```bash
# 检查节奏
forgeai review chapter 11 --agent pacing-checker

# 优化节奏
forgeai optimize chapter 11 --focus pacing
```

## 常见错误避免

### 1. 设定不充分

**错误表现：**
- 角色设定模糊
- 世界观不清晰
- 情节大纲缺失

**避免方法：**
- 写前完成所有设定
- 使用设定验证功能
- 定期检查设定一致性

### 2. 审查不及时

**错误表现：**
- 问题累积过多
- 后期修改工作量大
- 一致性问题严重

**避免方法：**
- 每章创作后立即审查
- 优先处理严重问题
- 定期进行全书检查

### 3. 备份不定期

**错误表现：**
- 数据丢失
- 无法回退到之前版本
- 创作历史丢失

**避免方法：**
- 定期创建备份
- 使用有意义的备份名称
- 导出项目完整备份

### 4. 质量不关注

**错误表现：**
- 追读力低
- 弃书风险高
- 读者体验差

**避免方法：**
- 定期检查追读力
- 关注质量评分
- 及时优化问题章节

## 进阶技巧

**技巧：**
- 为每条线维护独立上下文

**技巧：**

- 定期检查伏笔回收
- 确保所有悬念有解

### 4. 风格一致性维护

**技巧：**
- 使用风格配置文件
- 定期检查风格一致性
- 使用样章模板

## 总结

### 核心原则

1. **准备充分**：写前准备越充分，创作越顺利
2. **持续审查**：每章创作后立即审查，避免问题累积
3. **质量优先**：追读力和一致性比速度更重要
4. **定期备份**：重要节点创建备份，防止数据丢失
5. **灵活调整**：根据审查结果调整创作策略

### 推荐工作流

```bash
# 日常创作
forgeai write next              # 创作下一章
forgeai review last             # 审查最新章节
forgeai review fix last --auto  # 自动修复
forgeai status                  # 查看进度

# 每10章
forgeai review chapters 1-10 --all-agents  # 全面审查
forgeai report quality --chapters 1-10     # 质量报告
forgeai backup create --name "进度备份"    # 创建备份

# 完成后
forgeai review all --consistency            # 全书检查
forgeai report stats                        # 统计报告
forgeai export --formats markdown,docx,pdf  # 导出成稿
forgeai backup create --name "最终版本"     # 最终备份
```
