## 执行策略

基于 CONTEXT.md 中的决策，本阶段将按用户流程组织文档结构，采用分层文档策略。文档将分为4层：快速开始（极简）、详细指南（完整）、API参考（全参数）、示例库（真实案例）。

**核心原则:**
1. 新手能在5分钟内上手
2. 每个命令都有完整示例（输入、输出、预期结果）
3. Agent文档包含具体问题示例和修复建议
4. 提供完整的创作流程示例（从创建项目到完成第一章）

**文档结构:**

## Task 2.1: 完善命令文档

**Type:** docs
**Priority:** high
**Estimated time:** 1天

### 目标

创建完整的命令参考文档，覆盖所有15个核心命令，每个命令包含：
- 命令说明
- 完整参数列表
- 使用示例（基础、进阶、故障排查）
- 预期输出
- 常见问题

### 前置条件

- ✅ CLI命令已实现（system/scripts/forgeai.py）
- ✅ 现有命令文档模板（WRITE_COMMAND.md）

### 执行步骤

1. **创建命令文档目录结构**
   ```bash
   mkdir -p docs/03-命令参考
   ```

2. **按功能分组整理命令**
   - 核心命令: init, write, check
   - 审查命令: review, consistency
   - 管理命令: entity, foreshadow, timeline
   - 高级命令: analyze, search, context

3. **为每个命令创建文档**
   - 从 forgeai.py 提取命令定义
   - 从现有文档（WRITE_COMMAND.md）提取模板
   - 补充完整示例和输出

4. **创建命令速查表**
   - 单页PDF格式的命令速查表
   - 包含最常用命令和参数

### 验收标准

- [ ] 所有15个核心命令有完整文档
- [ ] 每个命令至少3个示例（基础、进阶、故障排查）

- [ ] 文档格式统一

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 文档过多导致维护困难 | 高 | 采用分层文档策略，核心文档优先 |
| 示例代码过时 | 中 | 每个版本发布前验证示例 |
| 文档与代码不一致 | 高 | 文档引用代码位置，便于更新 |
| 新手仍然难以理解 | 中 | 添加更多截图和视频教程链接 |
