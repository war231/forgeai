# REQUIREMENTS.md — ForgeAI

> **版本**: v1.0
> **最后更新**: 2026-04-19

---

## 需求概述

ForgeAI需要提供完整的AI辅助小说创作功能,包括样板书学习、自动化创作流程、多Agent审查、数据管理等核心能力。

---

## v1.0 核心需求

### REQ-001: 样板书拆解分析

**优先级**: Must
**状态**: ✅ 已完成

**描述**:
支持分析对标作品,提取结构、套路、爽点、文风等创作要素。

**验收标准**:
- [x] 支持TXT格式样板书导入
- [x] 提取章节结构(字数分布、节奏比例)
- [x] 识别爽点类型和密度
- [x] 提取主线套路和周期性套路
- [x] 分析文风特征(叙事视角、句长、对白密度)

**相关命令**:
```bash
forgeai analyze 样板书.txt -o ./样板书分析
```

---

**优先级**: Must
**状态**: ✅ 已完成

**描述**:
支持创建新的小说项目,自动生成目录结构和配置文件。

**验收标准**:
- [x] 自动创建标准目录结构(1-边界/2-设定/3-大纲/4-正文/5-审查)

- [x] 初始化数据库(index.db, vectors.db)

### REQ-003: 上下文提取

**优先级**: Must
**状态**: ✅ 已完成

**描述**:
为章节创作提供完整的上下文信息,包括前文、实体、伏笔、RAG检索结果。

**验收标准**:
- [x] 提取前5章正文内容
- [x] 提取活跃实体(最近10章)
- [x] 提取待回收伏笔
- [x] RAG语义检索相关片段
- [x] 支持主题指导(--query参数)
- [x] 支持智能上下文模式(--smart)

**相关命令**:
```bash

```

**相关命令**:

## 非功能需求

### NFR-001: 性能要求

- RAG检索响应时间 < 2秒
- 章节索引时间 < 5秒
- 实体提取时间 < 3秒
- 支持10万字以上小说项目

### NFR-002: 可用性要求

- 命令行界面友好
- 错误提示清晰
- 文档完善
- 示例丰富

### NFR-003: 可维护性要求

- 代码结构清晰
- 模块化设计
- 单元测试覆盖
- 文档同步更新

### NFR-004: 兼容性要求

- Python 3.8+
- Node.js 16+
- Windows/macOS/Linux
- VSCode 1.60+
