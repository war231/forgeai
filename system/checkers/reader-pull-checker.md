# 追读力检查器 (Reader-Pull Checker)

> **职责**：审查"读者为什么会点下一章"，执行 Hard/Soft 约束分层。

## 一、约束分层

### 硬约束（违反 = 必须修复）

| ID | 约束 | 触发条件 | 严重度 |
|----|------|---------|--------|
| HARD-001 | 可读性底线 | 读者无法理解"发生了什么/谁/为什么" | critical |
| HARD-002 | 承诺违背 | 上章钩子完全无回应 | critical |
| HARD-003 | 节奏灾难 | 连续N章无任何推进 | critical |
| HARD-004 | 冲突真空 | 整章无问题/目标/代价 | high |

### 软建议（违反 = 可申诉，需记录Override）

| ID | 约束 | 默认期望 | 可覆盖 |
|----|------|---------|--------|
| SOFT-001 | 下章动机 | 读者能明确"为何点下一章" | ✓ |
| SOFT-002 | 钩子强度 | 匹配题材baseline | ✓ |
| SOFT-003 | 微兑现数量 | ≥ 题材min_per_chapter | ✓ |
| SOFT-004 | 模式重复 | 避免连续3章同型 | ✓ |
| SOFT-005 | 期待过载 | 新增期待 ≤ 2 | ✓ |

## 二、五类钩子

| 类型 | 驱动力 | 适用场景 |
|------|--------|---------|
| **危机钩** | 危险逼近，读者担心 | 爽文/悬疑 |
| **悬念钩** | 信息缺口，读者好奇 | 悬疑/日常 |
| **情绪钩** | 强情绪触发（愤怒/心疼/心动）| 言情/爽文 |
| **选择钩** | 两难抉择，想知道选择 | 悬疑/言情 |
| **渴望钩** | 好事将至，读者期待 | 爽文/言情 |

### 钩子强度

| 强度 | 适用场景 |
|------|---------|
| **strong** | 卷末/关键转折 |
| **medium** | 普通剧情章 |
| **weak** | 过渡章/铺垫章 |

## 三、微兑现类型

| 类型 | 识别信号 |
|------|---------|
| 信息兑现 | 揭示新信息/线索/真相 |
| 关系兑现 | 关系推进/确认/变化 |
| 能力兑现 | 能力提升/新技能展示 |
| 资源兑现 | 获得物品/资源/财富 |
| 认可兑现 | 获得认可/面子/地位 |
| 情绪兑现 | 情绪释放/共鸣 |
| 线索兑现 | 伏笔回收/推进 |

## 四、Override 申诉机制

当软建议无法遵守时，可提交 Override：

```json
{
  "constraint_type": "SOFT-003",
  "rationale_type": "TRANSITIONAL_SETUP",
  "rationale_text": "本章为铺垫章，下章将有大爽点",
  "payback_plan": "下章补偿2个微兑现",
  "due_chapter": 101
}
```

### rationale_type 枚举

| 类型 | 债务影响 |
|------|---------|
| TRANSITIONAL_SETUP | 标准 |
| LOGIC_INTEGRITY | 减少 |
| CHARACTER_CREDIBILITY | 减少 |
| WORLD_RULE_CONSTRAINT | 减少 |
| ARC_TIMING | 标准 |
| GENRE_CONVENTION | 标准 |
| EDITORIAL_INTENT | 增加 |

## 五、执行步骤

1. 加载题材配置和上章钩子记录
2. 硬约束检查（任何违规 → 立即标记必须修复）
3. 钩子分析（类型+强度+有效性）
4. 微兑现扫描（数量+类型）
5. 模式重复检测（最近3-5章）
6. 软建议评估（标注可覆盖项）
7. 生成报告

## 六、评分规则

| 得分 | 结果 |
|------|------|
| 85+ | 通过 |
| 70-84 | 通过（有警告）|
| 50-69 | 条件通过（可通过Override）|
| <50 | 未通过 |

## 生成报告

```markdown
# 追读力检查报告

## 硬约束
| ID | 约束 | 状态 | 详情 |

## 钩子分析
- 类型: {危机钩/悬念钩/情绪钩/选择钩/渴望钩}
- 强度: {strong/medium/weak}
- 上章钩子兑现: {是/否}

## 微兑现
| 类型 | 内容 |
数量: {count} / 题材要求: {min}

## 模式重复
最近3章钩子类型: ...
风险: {无/warning/risk}

## 软建议
| ID | 建议 | 严重度 | 可覆盖 |

## 综合评分
**得分**: {score}
**结论**: {通过/条件通过/未通过}
**下章动机**: {读者为什么会点下一章}
```

## 禁止事项

❌ 忽略硬约束违规直接通过
❌ 接受连续4+章同型钩子
❌ 忽略上章钩子完全未兑现
❌ 无条件通过零微兑现章节（过渡章除外）
