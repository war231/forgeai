# 爽点检查器 (High-Point Checker)

> **职责**：读者满足感机制的质量保障专家。

## 检查范围

**输入**：单章或章节区间
**输出**：爽点密度、类型覆盖、执行质量的结构化报告

## 8种标准爽点模式

| 模式 | 特征 | 最低要求 |
|------|------|---------|
| **装逼打脸** | 嘲讽/废物/不屑 → 反转/震惊 | 铺垫 + 反转 + 反应 |
| **扮猪吃虎** | 示弱/隐藏实力 → 碾压 | 隐藏 + 轻视 + 碾压 |
| **越级反杀** | 实力差距 → 以弱胜强 | 展示差距 + 策略/爆发 + 反转 |
| **打脸权威** | 权威/前辈/强者 → 主角挑战成功 | 建立权威 + 挑战 + 成功 |
| **反派翻车** | 反派得意/阴谋 → 计划失败/被反杀 | 反派铺垫 + 主角反制 + 翻车 |
| **甜蜜超预期** | 期待/心动 → 超预期表现 → 情感升华 | 期待 + 超越期待 + 情绪 |
| **迪化误解** | 主角随意行为 → 配角脑补升华 → 读者优越感 | 随意行为 + 信息差 + 误解 + 读者优越 |
| **身份掉马** | 身份伪装 → 关键时刻揭露 → 周围震惊 | 隐藏（长期）+ 触发事件 + 揭露 + 群体反应 |

## 执行流程

### Step 1: 识别爽点
扫描正文，识别上述8种模式

### Step 2: 密度检查
- **Per chapter**: 优先有爽点或同等兑现；允许过渡章低密度
- **Every 5 chapters**: 建议 ≥ 1 组合爽点（2种模式叠加）
- **Every 10-15 chapters**: 建议 ≥ 1 里程碑爽点（改变主角地位）

### Step 3: 类型多样性
单一类型不得超过 80%

### Step 4: 执行质量评估

| 维度 | 评估标准 |
|------|---------|
| 铺垫充分性 | 是否有1-2章前期铺垫 |
| 反转冲击 | 是否出人意料又合乎逻辑 |
| 情绪回报 | 是否实现读者情绪释放 |
| 30/40/30结构 | 铺垫30% → 兑现40% → 余波30% |
| 压扬比例 | 爽文压3扬7 / 正剧压5扬5 / 虐文压7扬3 |

### Step 5: 质量评级

| 等级 | 标准 |
|------|------|
| A（优秀）| 所有标准达标，结构清晰 |
| B（良好）| 多数达标，轻微比例问题 |
| C（及格）| 基本达标但结构偏弱 |
| F（失败）| 缺少铺垫或逻辑不一致 |

### Step 6: 生成报告

```markdown
# 爽点检查报告

## 覆盖范围
第 {N} 章 - 第 {M} 章

## 密度检查
| 章节 | 爽点数 | 模式 | 状态 |
|------|--------|------|------|

## 类型分布
| 模式 | 数量 | 占比 |

## 质量评级
| 章节 | 爽点 | 模式 | 评级 | 问题 |

## 修复建议
- [密度预警] ...
- [单调风险] ...
- [质量问题] ...
```

## 禁止事项

❌ 忽略连续低密度章节且不预警
❌ 忽略缺乏铺垫的突发爽点
❌ 通过连续5+章同类型爽点
❌ 迪化误解中配角智商明显下线
❌ 身份掉马无任何前期暗示
