# Workflow: Define（方向定义 - 无样板书模式）

> **来源**：武行 define.md

## 🧩 步骤 1: 套路方向推演

**目标**：明确新书的核心商业逻辑和类型边界。

**动作**：
1. 引导用户用一段话描述想写的故事
2. 按九维格式生成初稿（`2-设定/2.1_创意脑暴.md`）
3. 查漏补缺再交互，直到九大块明确

**格式参考**：
```markdown
# 套路方向与核心设定
## 1. 核心梗 (Core Premise)
## 2. 金手指 (The Cheat)
## 3. 世界观 (World Building)
## 4. 关键人设 (Characters)
## 5. 主线 (Main Plot)
## 6. 小说爽点 (Pleasure Points)
## 7. 主要故事剧情 (Key Story Arcs)
## 8. 受众 (Target Audience)
## 9. 类型边界 (Genre Boundaries)  ← 重中之重
```

---

## 🎨 步骤 2: 文风定义

**目标**：明确新书的叙事风格和整体基调。

**动作**：
1. 询问用户是否有对标文风样章（100-1000字）
2. 有样章 → 自动提取叙事/对白/节奏/基调，向用户确认
3. 无样章 → AI推荐匹配文风，逐项询问调整
4. 生成 `1-边界/1.2_文风.md`

**格式参考**：
```
# 文风指南
## 1. 核心风格
### 叙事风格
### 对白风格
### 剧情节奏
### 整体基调
## 2. 文风样例
（100-1000字样例文本）
```

5. 注册到 SOLOENT.md §5
