# Workflow: Review（全流程动态质检）

> **融合设计**：武行主编审稿（定性）+ 六维Checker并行审查（定量）+ 进化式去AI味（迭代）
> **原则**：早发现，早治疗。设定服务于剧情，剧情服务于爽点。

---

## 🛠️ 模块 1: 设定质检

**触发**：完成 Ideation 流程后

**检查对象**：`2-设定/` 全部文件

**武行主编视角**：
1. 逻辑自洽性（力量体系闭环/金手指平衡/世界观支撑）
2. 爽点支撑度（金手指独特性/期待感设计）
3. 利益驱动逻辑（主角驱动力/反派智商）
4. 人物立体度（反差萌/记忆点）

---

## 📋 模块 2: 大纲质检

**触发**：完成 Outline 流程后

**检查对象**：`3-大纲/` 全部文件

**武行主编视角**：
1. 节奏密度（三章一小高潮/十章一大高潮/垃圾时间检测）
2. 期待感管理（钩子有效性/伏笔铺设）
3. 爽点逻辑（情绪释放/主角作用）
4. 一致性（设定对齐/战力检查）

---

## ✍️ 模块 3: 正文质检 + 六维并行审查

**触发**：完成 Write 流程后

**检查对象**：`4-正文/第X章_草稿.md`

### Step 1: 武行主编初审

1. **黄金三章法则**（仅开篇）：切入点/金手指时机/代入感
2. **沉浸感**：Show Don't Tell / 五感描写
3. **情绪调动**：爽点/憋屈点/装逼打脸
4. **文风与红线**：AI味检测 / 宪法检查
5. **完稿自检卡**：字数/大纲符合性/文风/接续/金手指合规

### Step 2: 六维并行审查

**按需选择**（不必每次全跑）：

| Checker | 何时用 | 读取文件 |
|---------|--------|---------|
| 爽点检查 | 每章 | `checkers/high-point-checker.md` |
| 一致性检查 | 每章 | `checkers/consistency-checker.md` |
| 节奏检查 | 每5-10章 | `checkers/pacing-checker.md` |
| OOC检查 | 每章 | `checkers/ooc-checker.md` |
| 连贯性检查 | 每章 | `checkers/continuity-checker.md` |
| 追读力检查 | 每章 | `checkers/reader-pull-checker.md` |

**推荐频率**：
```
每章必跑: 爽点 + 一致性 + OOC + 连贯性
每5章跑一次: 追读力
每10章跑一次: 节奏
```

### Step 3: 进化式去AI味

**触发**：Step 1-2 发现AI味问题时

读取 `skills/humanize.md`，执行 baseline→challenger→score 迭代流程：

1. 双引擎打分（网文AI味60% + 通用AI味40%）
2. 生成3个挑战者（自然版/平衡版/规则驱动版）
3. 评分竞争，保留更像人的版本
4. 必要时修复轮（最多3轮）
5. 输出最终版本 + 优化过程报告

### Step 4: 综合评审

汇总所有审查结果：

```markdown
# 综合审查报告

## 武行主编初审
- 结论: {通过/需修改}
- 问题: ...

## 六维审查
| 维度 | 评分 | 状态 |
|------|------|------|
| 爽点密度 | A | ✓ |
| 设定一致性 | B | ⚠️ 1处需修复 |
| 节奏平衡 | A | ✓ |
| OOC检测 | A | ✓ |
| 叙事连贯 | B | ⚠️ 1处需修复 |
| 追读力 | 82 | 通过(有警告) |

## 去AI味优化
- 优化前: 67.8 (C)
- 优化后: 87.2 (A)
- 提升: +19.4分

## 最终结论
- 状态: {通过/需修改/需重写}
- 优先修复项: ...
```
