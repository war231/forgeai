# 核心约束：防幻觉三定律

> **定位**：此文件为章节创作时的核心约束检查，确保三大定律精确执行。
>
> **原则**：所有约束为"强制性"，不做软性建议。

---

## 三大定律（低自由度 - 必须精确执行）

| 定律 | 规则 | 检查方式 | 违反后果 |
|------|------|----------|----------|
| **大纲即法律** | 严格执行大纲，不得擅自发挥 | 审查时对照大纲 | 阻断重写 |
| **设定即物理** | 实力/招式/物品 ≤ index.db 记录 | 写作前查询确认 | 自动修正 |
| **发明需识别** | 新实体由 Data Agent 自动提取 | 章节完成后处理 | 标记待确认 |

---

## 定律一：大纲即法律

### 核心规则
- 章节内容必须严格遵循大纲规划
- 不得擅自增加/删除/修改大纲中的关键事件
- 不得改变大纲中设定的人物关系、时间线、因果关系

### 检查清单
```
□ 本章事件是否在大纲中有明确规划？
□ 人物行为是否符合大纲设定？
□ 时间线是否与大纲一致？
□ 有无擅自添加的支线剧情？
□ 有无遗漏大纲要求的事件？
```

### 违反示例
```
❌ 大纲规划：主角在擂台上击败对手
   实际写作：主角直接认输，改日再战

❌ 大纲规划：女主发现男主身份
   实际写作：男主主动坦白身份
```

### 正确处理
```
✅ 发现大纲不合理时：
   1. 先按大纲写完
   2. 提交"大纲修订建议"
   3. 等待用户确认后修改
```

---

## 定律二：设定即物理

### 核心规则
- 角色实力不得超过 index.db 记录
- 招式/物品/能力必须在 index.db 中存在或有获取途径
- 世界观规则（修炼体系、社会规则）不得违反

### 检查清单
```
□ 主角当前实力是否与 index.db 一致？
□ 使用的招式是否已学习/获得？
□ 持有的物品是否已获取？
□ 实力变化是否有明确的突破/获得场景？
□ 世界观设定是否遵守？
```

### 违反示例
```
❌ index.db 显示：主角筑基期
   实际写作：主角一掌拍死金丹期（无突破场景）

❌ index.db 显示：主角未学习"天雷掌"
   实际写作：主角突然使用天雷掌击败敌人
```

### 正确处理
```
✅ 需要新能力时：
   1. 先写获取场景（拜师/领悟/传承）
   2. Data Agent 自动更新 index.db
   3. 后续章节正常使用

✅ 需要实力突破时：
   1. 先写突破场景
   2. 更新 index.db
   3. 后续章节展示新实力
```

## 定律三：发明需识别

### 核心规则
- 新角色、新地点、新物品、新招式可以自由创作
- 但必须由 Data Agent 自动识别并写入 index.db
- 不确定实体会标记为 `uncertain`，需人工确认

### 新实体处理流程

```
写作时    → 直接写纯正文，新实体正常描写
完成后    → Data Agent 自动识别新实体
识别结果  → 写入 index.db
不确定项  → 标记 uncertain，人工确认
```

### 可自动识别的实体类型
| 类型 | 识别规则 | 示例 |
|------|----------|------|
| 角色 | 有名字、有对话、有行为 | "叶良辰冷笑道..." |
| 地点 | 有名称、有描写 | "天云宗，云雾缭绕..." |
| 物品 | 有名称、有功能描述 | "天雷掌秘籍..." |
| 招式 | 有名称、有效果描述 | "一记烈焰掌轰出..." |
| 组织 | 有名称、有成员 | "天剑门三大长老..." |

### 需人工确认的情况
```
⚠️ 实体名称模糊（"那个老人"、"黑衣人"）
⚠️ 实体类型不确定（是人名还是地名？）
⚠️ 与现有实体可能冲突（同名不同人？）
```

## 章节约束分层

### Hard（必须执行）
- 本章可读性达标：读者能回答"发生了什么 / 谁在做什么 / 为什么"
- 本章必须存在清晰推进：问题、目标、代价、关系变化、信息变化至少一项
- 若上章有明确承诺（钩子/未闭合问题），本章必须回应
- 禁止输出占位正文（`[待补充]`、`[TODO]`、`...（省略）...`）

### Soft（建议执行）
- 开头尽早进入冲突/风险/强情绪（建议前 200-400 字）
- 章末建议保留未闭合问题或期待锚点
- 局面变化保持节奏感（参考 800-1400 字一个脉冲）

### Style（可选强化）
- 对话尽量带意图（试探/回避/施压/诱导）
- 避免连续大段纯解释
- 避免"回去休息了"式机械收尾

## 禁止事项清单

| 禁止项 | 说明 | 替代方案 |
|--------|------|----------|
| `[待补充]` | 占位符 | 必须完整写出 |
| `[TODO]` | 占位符 | 必须完整写出 |
| `...（省略）...` | 跳过内容 | 必须完整写出 |
| 无突破的实力飞跃 | 违反设定即物理 | 先写突破场景 |
| 跳过大纲事件 | 违反大纲即法律 | 严格按大纲执行 |
| 战斗后无善后 | 都市异能题材特有 | 补充善后描写 |

## 错误处理流程

```
发现违反 → 记录问题类型 → 阻断/警告 → 提供修复建议

违反类型：
├── Critical（阻断）
│   ├── 大纲重大偏离
│   ├── 设定严重冲突
│   └── 占位符未填写
│
├── Warning（警告）
│   ├── 实力小幅度不一致
│   ├── 新实体未识别
│   └── 时间线小问题
│
└── Info（提示）
    ├── 软约束未满足
    └── 风格建议
```
