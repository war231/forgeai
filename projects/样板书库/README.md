# 样板书库

> 存放用于参考学习的样板书原文

---

## 📂 目录结构

```
样板书库/
├── 凡人修仙传/              ← 小说名文件夹
│   ├── 第1章.txt
│   ├── 第2章.txt
│   └── ...
├── 遮天/                    ← 另一部小说
│   ├── 第1章.txt
│   └── ...
└── README.md
```

---

## 🎯 使用方法

### 1. 创建小说文件夹

在 `样板书库/` 下创建以小说名命名的文件夹：

```
样板书库/凡人修仙传/
样板书库/遮天/
样板书库/斗破苍穹/
```

### 2. 放置样板书章节

将样板书章节文件放入对应文件夹：

- **格式**：`.txt` 文件
- **编码**：UTF-8
- **命名**：`第1章.txt`、`第2章.txt` 或合并为一个文件

### 3. 调用 book-analyzer 分析

在小说项目中调用 `book-analyzer` 技能时，指定样板书路径：

```
样板书路径：../样板书库/凡人修仙传/
```

---

## 📋 建议的样板书数量

| 分析目的 | 建议章节数 |
|---------|-----------|
| 快速了解套路 | 10-20 章 |
| 深度分析节奏 | 30-50 章 |
| 全面研究文风 | 50-100 章 |

---

## ⚠️ 注意事项

1. **版权**：样板书仅供个人学习参考，不得用于商业用途
2. **Token 消耗**：章节越多，分析消耗 Token 越多
3. **编码**：必须是 UTF-8 编码，否则中文会乱码

---

## 🔄 分析结果存储

分析结果会存储在**小说项目**的 `1-边界/样板书分析/` 目录：

```
projects/你的小说/
└── 1-边界/
    └── 样板书分析/        ← 分析结果
        ├── 分析报告.md
        ├── 套路提取.md
        └── ...
```

样板书原文保留在 `样板书库/`，不污染小说项目目录。
