# 金手指设定

## 1. 核心概念

| 维度 | 内容 |
|-----|------|
| **本质** | 穿越者的历史知识库 |
| **来源** | 前世是历史系研究生，熟读《三国志》《后汉书》《资治通鉴》 |
| **独有性** | 无人能预知未来，只有主角知道"剧本" |

---

## 2. 功能模块

### 2.1 基础功能（1.0）

| 功能 | 说明 | 示例 |
|-----|------|------|
| **大事预知** | 知道重大历史事件走向 | 吕布必败、袁绍必亡、赤壁曹操必败 |
| **人物命运** | 知道历史人物的生死结局 | 郭嘉会死、关羽会死、曹操会死 |

### 2.2 进阶功能（2.0）

| 功能 | 说明 | 示例 |
|-----|------|------|
| **细节推演** | 知道事件的具体细节 | 许攸会来投、乌巢粮草位置、黄盖诈降 |
| **时机把握** | 知道事件发生的时间 | 官渡之战何时爆发、赤壁之战何时开战 |

### 2.3 终极功能（3.0）

| 功能 | 说明 | 代价 |
|-----|------|------|
| **改变历史** | 主动干预，改变历史走向 | 后续记忆失效，金手指"失灵" |

---

## 3. 平衡性设计

### 3.1 消耗

| 维度 | 内容 |
|-----|------|
| **直接消耗** | 无 |
| **间接消耗** | 每次"预言"都增加被怀疑风险 |

### 3.2 限制

| 限制 | 说明 |
|-----|------|
| **记忆模糊** | 正史vs演义vs真实历史有出入，主角记忆可能有偏差 |
| **蝴蝶效应** | 改变太多会导致后续不可预测，金手指失效 |
| **伪装需求** | 必须装作"推演得出"，不能说"我看过史书" |

### 3.3 副作用

| 副作用 | 说明 |
|-----|------|
| **记忆失效** | 改变历史后，后续记忆逐渐模糊 |
| **身份危机** | 预言太准会被怀疑"妖言惑众"或"别有用心" |

---

## 4. 进化路线

```
阶段1：被动预知（1-30章）
├─ 只知道大事走向
├─ 被动应对，偶尔"随口说中"
└─ 风险：被怀疑但未暴露

阶段2：主动推演（31-150章）
├─ 能推演事件细节
├─ 主动建议，获得信任
└─ 风险：被郭嘉/荀彧深度审视

阶段3：改变历史（151-200章）
├─ 尝试改变关键事件（救郭嘉、阻止赤壁）
├─ 金手指逐渐失效
└─ 风险：后续不可预测，必须靠自己
```

---

## 5. 视觉表现

| 维度 | 内容 |
|-----|------|
| **UI界面** | 无系统界面，纯靠记忆 |
| **具象化形态** | 主角脑海中闪过"史书记载"片段 |
| **触发方式** | 看到相关人物/事件时自动触发记忆 |

---

## 6. 使用示例

### 示例1：预言吕布必败

```
【触发】曹操问："吕布如何？"
【记忆】史书记载：建安三年，曹操围下邳，吕布部将叛变，擒吕布降。
【伪装】"某以为，吕布勇而无谋，必败于内。"
【结果】曹操点头，郭嘉脑补："此人深不可测！"
```

### 示例2：预言官渡之战

```
【触发】曹操问："袁绍势大，如何是好？"
【记忆】史书记载：许攸献计烧乌巢，袁绍大败。
【伪装】"某闻袁绍粮草屯于乌巢，若能烧之..."
【结果】曹操采纳，郭嘉脑补："此人竟能推演至此！"
```

### 示例3：改变历史（救郭嘉）

```
【触发】郭嘉病重
【记忆】史书记载：郭嘉病逝于征乌桓途中。
【行动】建议曹操勿征乌桓，或派医者随行
【结果】郭嘉存活？后续记忆失效，金手指失灵
```
